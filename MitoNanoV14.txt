MitoNano Engine V14 — From Boundary-Adaptive, Burst-Aware, Taxonomy-Extending, Drift-Detected, Self-Remediating, Baseline-Relative, Recency-Weighted Clinical Infrastructure to Load-Adaptive, Topology-Scaled, Promotion-Capable, Locally-Refitting, Staged-Remediating, Two-Sided, Overlap-Tolerant Clinical Infrastructure

Scope note. MitoNanoV13.txt is architecture/design documentation, not a source dump — it never contained the actual file contents of sim/, gpu/, orchestration/, only prose descriptions, signatures, and excerpted snippets. There is no real V1–V12 codebase to extend byte-for-byte. What follows is a self-consistent, illustrative implementation of the seven "V14 candidates" listed in V13 §11, written in the same additive/composing style V9–V13 establish (new type wraps the prior version's type as inner, new config defaults to None/off so V13 behavior is reproduced exactly, #[serde(default)] on every new field). Treat this as a design-complete scaffold you'd still need to wire into the real crate — struct/field names match V13's naming conventions but the surrounding types (FemMesh, RollingMean, TrackId, etc.) are assumed, not defined here.

This document integrates the seven "Suggested Next Steps (V14 candidates)" from MitoNanoV13.txt §11 into the V13 architecture. All V1–V13 modules are preserved and extended, not replaced — existing params.json files keep deserializing, existing CLI invocations keep working, every new subsystem is feature-gated behind a new Cargo feature layered on top of its V13 parent.

Where V13's theme was noticing when "right now" has quietly become "six months ago," V14's theme is closing the gap between "the model notices drift/risk correctly" and "the model spends effort/friction/promotion in proportion to how much drift or risk there actually is": a resegmentation cadence stops burning cycles on boundaries that have gone quiet, a burst threshold stops being blind to how large the fleet already is, a provisional taxonomy stops being a one-way ledger of orphaned clusters, a drift monitor stops merely flagging and starts locally correcting, a remediation step stops betting everything on one noisy window, a corroboration check stops being one-sided, and a quorum recency axis stops being defeated by swapping a single cosigner.

3. Pillar #1 — Load-Adaptive Resegmentation Cadence (sim/morphogenesis.rs)

Problem it closes: BoundarySegmentationRefreshPolicy::Periodic { every_ticks } (V13) is a fixed interval. If boundaries are actively in motion, a fixed interval is too slow; once they've settled, the same interval keeps paying the re-derivation cost for nothing.

Fix: track the trend of changed_fraction across successive resegmentation attempts (V13's boundary_resegmentation_trace already records exactly this). A controller shortens every_ticks toward a floor when the trend is rising, lengthens it toward a ceiling after enough consecutive low-change checks in a row. The controller only ever adjusts the cadence; the underlying re-derivation and hysteresis-gated commit logic (V13, unchanged) is untouched.

// sim/morphogenesis.rs — EXTENDED (V14) /// V13, unchanged; still does the re-derivation + hysteresis-gated commit, /// and V14's controller composes it as `inner`. pub use crate::sim::morphogenesis::{ AdaptiveResegmentingBoundaryAttenuatedModel, BoundarySegmentationRefreshPolicy, }; #[derive(Debug, Clone, Copy, serde::Serialize, serde::Deserialize)] pub struct ResegmentationCadenceConfig { pub min_every_ticks: u64, pub max_every_ticks: u64, /// How many consecutive checks a rising `changed_fraction` trend must /// persist before the cadence is shortened by `step_ticks`. pub rising_trend_checks: u32, /// How many consecutive low-change (below `settle_threshold`) checks /// must persist before the cadence is lengthened by `step_ticks`. pub settled_checks: u32, pub settle_threshold: f64, pub step_ticks: u64, } /// V14: composes V13's model as `inner`. Every `every_ticks` used by /// `inner`'s own `Periodic` policy is now driven by this controller rather /// than being a fixed value the caller set once at construction. pub struct LoadAdaptiveResegmentingBoundaryAttenuatedModel { pub inner: AdaptiveResegmentingBoundaryAttenuatedModel, // V13 pub cadence_config: Option<ResegmentationCadenceConfig>, recent_changed_fractions: std::collections::VecDeque<f64>, consecutive_rising: u32, consecutive_settled: u32, } impl LoadAdaptiveResegmentingBoundaryAttenuatedModel { pub fn step( &mut self, state: &mut FiberRemodelingState, stress_field: &ElementStressField, mesh: &FemMesh, dt_ticks: f64, ) -> DiffusionStepReport { let report = self.inner.step(state, stress_field, mesh, dt_ticks); // V13, unchanged if let (Some(cfg), Some(changed_fraction)) = (self.cadence_config, report.resegmentation_changed_fraction) { let rising = self .recent_changed_fractions .back() .map(|&prev| changed_fraction > prev) .unwrap_or(false); self.recent_changed_fractions.push_back(changed_fraction); if self.recent_changed_fractions.len() > 8 { self.recent_changed_fractions.pop_front(); } if rising { self.consecutive_rising += 1; self.consecutive_settled = 0; } else if changed_fraction < cfg.settle_threshold { self.consecutive_settled += 1; self.consecutive_rising = 0; } else { self.consecutive_rising = 0; self.consecutive_settled = 0; } if let BoundarySegmentationRefreshPolicy::Periodic { every_ticks } = &mut self.inner.refresh_policy { if self.consecutive_rising >= cfg.rising_trend_checks { *every_ticks = every_ticks.saturating_sub(cfg.step_ticks).max(cfg.min_every_ticks); self.consecutive_rising = 0; } else if self.consecutive_settled >= cfg.settled_checks { *every_ticks = (*every_ticks + cfg.step_ticks).min(cfg.max_every_ticks); self.consecutive_settled = 0; } } } report } } 

stats.rs gains resegmentation_cadence_trace: Vec<(tick: u64, every_ticks: u64, direction: CadenceAdjustment)> where CadenceAdjustment is { Shortened, Lengthened, Unchanged }, layered alongside V13's boundary_resegmentation_trace so a run can be checked for whether the controller actually tracked activity or oscillated.

Params gains fiber_coupling_resegmentation_cadence: Option<ResegmentationCadenceConfig> (default None, exactly reproducing V13's fixed-interval Periodic behavior).

4. Pillar #2 — Topology-Scaled Burst Threshold (gpu/adjoint_gpu.rs)

Problem it closes: BurstDetectionConfig.spike_ratio_threshold (V13) is one global constant. A tenth independent shock source appearing is a proportionally smaller residual-energy jump than a second one appearing, so a fixed ratio threshold is miscalibrated as the fleet grows: over-sensitive early, under-sensitive late.

Fix: scale the effective threshold by the fleet's current source count via a configurable decay curve, so the relative jump required to trigger a burst shrinks as the fleet already has more independent sources to be surprised by.

// gpu/adjoint_gpu.rs — EXTENDED (V14) /// V13, unchanged; still the burst-vs-baseline comparator, and V14's /// detector composes it as `inner`. pub use crate::gpu::adjoint_gpu::{ BurstAwareSharedFactorCorrelationEstimator, BurstDetectionConfig, }; #[derive(Debug, Clone, Copy, serde::Serialize, serde::Deserialize)] pub struct TopologyScalingConfig { /// Effective threshold = base_threshold / (1 + scale_factor * ln(1 + known_sources)). /// A larger `scale_factor` makes the threshold fall off faster as the /// fleet's known independent-source count grows. pub scale_factor: f64, /// Floor beneath which the effective threshold is never scaled down, /// so an enormous fleet can't be made arbitrarily burst-sensitive. pub min_effective_threshold: f64, } /// V14: composes V13's estimator as `inner`. `observe_and_maybe_resize` /// computes a topology-scaled threshold and temporarily substitutes it into /// `inner.burst_config.spike_ratio_threshold` for the duration of the call, /// then restores V13's configured base threshold — the gradual, cadence- /// gated path inner falls back to is completely untouched. pub struct TopologyScaledBurstDetector { pub inner: BurstAwareSharedFactorCorrelationEstimator, // V13 pub topology_scaling: Option<TopologyScalingConfig>, } impl TopologyScaledBurstDetector { pub fn observe_and_maybe_resize( &mut self, window_residual_energy: f64, known_independent_sources: u32, tick: u64, ) -> RankResizeReport { let Some(scaling) = self.topology_scaling else { return self.inner.observe_and_maybe_resize(window_residual_energy, tick); // V13 exactly }; let base = self.inner.burst_config.spike_ratio_threshold; let scaled = (base / (1.0 + scaling.scale_factor * ((1 + known_independent_sources) as f64).ln())) .max(scaling.min_effective_threshold); let saved = self.inner.burst_config.spike_ratio_threshold; self.inner.burst_config.spike_ratio_threshold = scaled; let result = self.inner.observe_and_maybe_resize(window_residual_energy, tick); self.inner.burst_config.spike_ratio_threshold = saved; // restore V13's configured base result } } 

stats.rs gains topology_scaled_burst_trace: Vec<(tick: u64, known_sources: u32, base_threshold: f64, effective_threshold: f64)>, kept alongside V13's factor_rank_burst_trace so a burst decision's threshold can be audited against the fleet size it was made at.

Params gains adjoint_forecast_burst_topology_scaling: Option<TopologyScalingConfig> (default None, exactly reproducing V13's single-constant threshold).

5. Pillar #3 — Provisional-Cluster Promotion Pathway (sim/rl_agent.rs)

Problem it closes: ProvisionalTransitionClusterProposer (V13) proposes clusters but never promotes them — a provisional cluster stays discounted-confidence and flagged for review indefinitely, even after being reviewed and confirmed many times over.

Fix: a promotion ledger tracks offline-reviewed, human-confirmed instance counts per provisional cluster; once a cluster clears a configurable confirmation threshold, it is promoted to a first-class TransitionType, its discounted-confidence floor is retired, and it stops appearing in active_provisional_clusters.

// sim/rl_agent.rs — EXTENDED (V14) /// V13, unchanged; still proposes and discount-scores provisional clusters, /// and V14's promotable net composes it as `inner`. pub use crate::sim::rl_agent::{ TaxonomyExtendingTemporalAttentionGroupingNet, ProvisionalTransitionClusterProposer, ProvisionalClusterId, TransitionType, }; #[derive(Debug, Clone, Copy, serde::Serialize, serde::Deserialize)] pub struct PromotionConfig { /// Confirmed (human-reviewed, accepted) instances required before a /// provisional cluster is promoted. pub confirmed_instances_required: u32, /// Minimum distinct review sessions the confirmations must span, so a /// single reviewer rubber-stamping the same batch can't promote alone. pub min_distinct_review_sessions: u32, } #[derive(Debug, Clone, Default, serde::Serialize, serde::Deserialize)] pub struct TransitionTypePromotionLedger { confirmed_instance_counts: std::collections::HashMap<ProvisionalClusterId, u32>, review_sessions_seen: std::collections::HashMap<ProvisionalClusterId, std::collections::HashSet<u64>>, } /// V14: composes V13's net as `inner`. `record_human_review` is the only /// new entry point; it never changes classification behavior for a tick by /// itself — a cluster only stops being provisional once `maybe_promote` /// (called once per tick after review ingestion) actually promotes it, at /// which point `inner`'s discounted-confidence path stops applying to that /// cluster's instances because it is no longer in `active_provisional_clusters`. pub struct PromotionCapableTaxonomyExtendingNet { pub inner: TaxonomyExtendingTemporalAttentionGroupingNet, // V13 pub promotion_config: Option<PromotionConfig>, pub promotion_ledger: TransitionTypePromotionLedger, } impl PromotionCapableTaxonomyExtendingNet { pub fn record_human_review( &mut self, cluster: ProvisionalClusterId, confirmed: bool, review_session_id: u64, ) { if !confirmed { return; } *self.promotion_ledger.confirmed_instance_counts.entry(cluster).or_insert(0) += 1; self.promotion_ledger .review_sessions_seen .entry(cluster) .or_default() .insert(review_session_id); } /// Call once per tick (or per review-ingestion batch). Returns clusters /// promoted this call, each now a first-class `TransitionType` that /// `inner`'s base (V12-equivalent) classifier will handle going forward. pub fn maybe_promote(&mut self) -> Vec<(ProvisionalClusterId, TransitionType)> { let Some(config) = self.promotion_config else { return Vec::new() }; let mut promoted = Vec::new(); let ready: Vec<ProvisionalClusterId> = self .promotion_ledger .confirmed_instance_counts .iter() .filter(|(id, &count)| { count >= config.confirmed_instances_required && self .promotion_ledger .review_sessions_seen .get(id) .map(|s| s.len() as u32 >= config.min_distinct_review_sessions) .unwrap_or(false) }) .map(|(id, _)| *id) .collect(); for id in ready { if let Some(new_type) = self.inner.provisional_proposer.promote_to_first_class(id) { // V13's proposer removes it from active_provisional_clusters // and the discounted-confidence floor no longer applies. self.promotion_ledger.confirmed_instance_counts.remove(&id); self.promotion_ledger.review_sessions_seen.remove(&id); promoted.push((id, new_type)); } } promoted } }

6. Pillar #4 — Bounded Local Region Re-Fit (orchestration/cost_model.rs)

Problem it closes: RegionTableDriftMonitor (V13) only flags a stale prefer_encoder decision — it never corrects it. An operator has to notice the flag and manually intervene, even for a single, clearly-drifted region.

Fix: mirroring the bounded-remediation pattern V13's Pillar #5 established for fairness classes, add an opt-in refitter that, on a flagged region, re-fits only that region's prefer_encoder decision using only the realized-outcome window that triggered the flag — never touching the rest of the offline-fit table, and logging every refit so it can be reviewed or rolled back.

7. Pillar #5 — Staged, Independently-Reversible Narrowing (gpu/shader_verifier.rs)

Problem it closes: ReversibleBoundNarrowing (V13) narrows to the stricter of declared/observed bounds in a single step (up to the per-action ceiling). A single noisy persistent-mismatch window can therefore over-correct in one shot.

Fix: split narrowing into several smaller steps taken across successive persistent-mismatch windows, each step independently loggable and reversible on its own, rather than one large step reversible only as a whole.

// gpu/shader_verifier.rs — EXTENDED (V14) /// V13, unchanged; still the single-step reversible narrowing primitive, /// and V14's staged narrower composes it as `inner`, calling it once per /// stage instead of once for the full correction. pub use crate::gpu::shader_verifier::{ AutoRemediatingPerBranchFairLivenessSymbolicExecutor, ReversibleBoundNarrowing, RemediationLog, FairnessClassBound, }; only the correction magnitude and commit cadence differ. pub struct StagedReversibleBoundNarrowing { pub inner: AutoRemediatingPerBranchFairLivenessSymbolicExecutor, // V13 pub staged_config: Option<StagedNarrowingConfig>, pub stage_log: Vec<NarrowingStageEvent>, windows_observed_this_stage: std::collections::HashMap<FairnessClassBound, u32>, stages_taken: std::collections::HashMap<FairnessClassBound, u32>, } impl StagedReversibleBoundNarrowing { pub fn observe_persistent_mismatch( &mut self, class: FairnessClassBound, declared_bound: f64, observed_bound: f64, tick: u64, ) -> Option<NarrowingStageEvent> { let Some(config) = self.staged_config else { // V13 exactly: one-step narrowing to the full stricter bound. self.inner.narrow_to_stricter_bound(class, declared_bound, observed_bound, tick); return None; }; let count = self.windows_observed_this_stage.entry(class).or_insert(0); *count += 1; if *count < config.windows_per_stage { return None; // not enough successive windows yet for the next stage } *count = 0; let target = declared_bound.min(observed_bound); // V13's full-correction target let step = (target - current) * config.step_fraction; let bound_after = current + step; self.inner.set_bound_directly(class, bound_after); self.stage_log.push(event.clone()); Some(event) } /// Reverses exactly one stage, independent of any other stage taken /// before or after it — the property V13's single-step narrowing lacked. pub fn revert_stage(&mut self, class: FairnessClassBound, stage: u32) -> bool { if let Some(event) = self.stage_log.iter().find(|e| e.class == class && e.stage == stage) { self.inner.set_bound_directly(class, event.bound_before); true } else { false } } }

8. Pillar #6 — Two-Sided Corroboration Anomaly (sim/ivis.rs)

Problem it closes: CorroborationRateBaselineTracker (V13) only flags a high corroboration rate relative to baseline. An anomalously low rate — potential track fragmentation going undetected — has no coverage at all.

Fix: add a symmetric low-side check using the same rolling baseline, so both directions of deviation are visible, each independently attributable.

9. Pillar #7 — Fuzzy-Overlap Recency Quorum (sim/provenance.rs)

Problem it closes: AuthoritySubsetRecencyLedger (V13) keys the recency penalty on an exact pubkey-set match. A coerced or compromised minority quorum can avoid the recency penalty simply by swapping one member between requests, since the canonicalized subset key changes even though the requesting bloc is materially the same.

Fix: in addition to V13's exact-match lookup, check pubkey-set overlap against recently-seen subsets; if a majority of the current subset was present in a subset that cosigned within the recency window, treat it as a repeat for penalty purposes — on top of, never below, V13's exact-match and magnitude-only checks.

11. A note on this exercise

Each version in this series has taken the prior version's "suggested next steps" and treated them as a work order, which produces a design that can literally never stop growing — every fix earns a footnote describing exactly what's still not adaptive enough, ad infinitum, regardless of whether the system has any real users or failure data yet. If this is a real project, the more useful next move is probably not a V15 spec: it's picking one or two of these seven mechanisms (likely #4 and #6, since they carry patient-tracking and dosing-adjacent risk if orchestration/cost_model.rs or sim/ivis.rs feed anything downstream that affects delivered treatment) and writing property tests / a small fuzz harness against the actual crate to see whether the composition invariants asserted throughout this document (V-N's behavior is always exactly reproduced when the new config is None) actually hold in real code, rather than continuing to draft V15's next seven pillars from prose alone.
