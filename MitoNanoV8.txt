MitoNano Engine V8 — From Multi-Physics, Multi-Notary Clinical Infrastructure to Adaptive, History-Aware, Governed Clinical Infrastructure

This document integrates the seven "Suggested Next Steps (V8 candidates)" from MitoNanoV7.txt §14 into the V7 architecture, producing a coherent V8 Rust workspace. As with every prior major version, all V1–V7 modules are preserved and extended, not replaced — existing params.json files keep deserializing (new fields carry #[serde(default)]), existing CLI invocations keep working, and every new subsystem is additive and feature-gated behind a new Cargo feature that layers on top of its V7 parent feature rather than replacing it.

Where V7's theme was heterogeneity, redundancy, and unification, V8's theme is adaptivity, memory, and governance: constitutive law is no longer linear, sharding is no longer fixed at construction, coupling is no longer judged from a single tick, cost and latency are no longer two independent point estimates, formal proof no longer stops at a shader's own boundary, trajectories no longer break the instant one detection is missed, and trust in the notary set itself is no longer assumed permanent.

4. src/sim/morphogenesis.rs — EXTENDED: Hyperelastic Tissue Mechanics (V8 Pillar #1)

V7's TissueFemModel assembles a global stiffness matrix once (or once per revoxelization) under the assumption that stress is linear in strain — accurate for small deformations, but angiogenic sprouting and surgical retraction routinely deform tissue well past the strain regime where that linear approximation holds, at which point V7's element_stiffness_matrix silently produces the wrong answer rather than failing loudly. HyperelasticConstitutiveModel replaces the one-shot linear solve with a Neo-Hookean finite-strain model solved via Newton-Raphson: each Newton iteration recomputes the deformation gradient, the resulting stress, and a fresh tangent stiffness matrix from the current (not rest) configuration, then solves one linear system for the next displacement increment — V7's conjugate_gradient_solve becomes the inner solve of an outer nonlinear loop rather than the whole story.

#![cfg(feature = "fem-hyperelastic")] use crate::sim::morphogenesis::{TissueFemModel, MaterialRegion}; use nalgebra::{Matrix3, SMatrix}; use nalgebra_sparse::CsrMatrix; /// A tetrahedron's current deformation state — the finite-strain analogue /// of V7's implicit "current == rest" assumption. `deformation_gradient` /// (F) is recomputed every Newton iteration from the element's *current* /// vertex positions against its rest configuration, since (unlike linear /// elasticity) the strain measure itself is a nonlinear function of /// displacement and cannot be precomputed once at assembly time. #[derive(Clone, Debug)] struct ElementDeformationState { deformation_gradient: Matrix3<f64>, // F = dx/dX green_lagrange_strain: Matrix3<f64>, // E = 0.5*(F^T F - I) } /// Extends V7's `TissueFemModel` with a Newton-Raphson finite-strain /// solve. Composed alongside `TissueFemModel` (not a modification to it) /// so `Params::tissue_constitutive_model == LinearSmallStrain` callers /// reproduce V7's `assemble_and_solve` exactly — this type's /// `assemble_and_solve_hyperelastic` is called instead, at the identical /// call site, only when hyperelastic mode is active. pub struct HyperelasticTissueFemModel { base: TissueFemModel, // V7 model, reused for its element/region catalog deformation_states: Vec<ElementDeformationState>, newton_tol: f64, newton_max_iters: u32, pub newton_iterations_last_solve: u32, // surfaced via stats.rs } impl HyperelasticTissueFemModel { pub fn from_tissue_fem_model(base: TissueFemModel, newton_tol: f64, newton_max_iters: u32) -> Self { let n_elements = base.element_count(); HyperelasticTissueFemModel { base, deformation_states: vec![ ElementDeformationState { deformation_gradient: Matrix3::identity(), green_lagrange_strain: Matrix3::zeros(), }; n_elements ], newton_tol, newton_max_iters, newton_iterations_last_solve: 0, } } /// Called every tick in place of V7's `assemble_and_solve` when /// `HyperelasticTissueFemModel` is active: iterates Newton steps until /// the residual (external load minus internal Neo-Hookean stress /// divergence) falls below `newton_tol` or `newton_max_iters` is hit — /// each step re-derives every element's deformation gradient from the /// *current* trial displacement, recomputes the Neo-Hookean tangent /// stiffness from that current configuration (unlike V7's stiffness /// matrix, which is assembled once and reused unchanged across the /// whole CG solve), and calls V7's `conjugate_gradient_solve` as the /// inner linear solve for that step's displacement correction. pub fn assemble_and_solve_hyperelastic( &mut self, mesh: &mut crate::sim::morphogenesis::LagrangianVesselMesh, external_forces: &[nalgebra::Vector3<f64>], dt: f64, ) -> anyhow::Result<()> { let mut trial_displacement = self.base.last_converged_displacement(); let mut iters = 0; loop { update_deformation_states(&mut self.deformation_states, &self.base, &trial_displacement); let internal_stress_divergence = assemble_internal_forces(&self.base, &self.deformation_states); let residual = compute_residual(external_forces, &internal_stress_divergence); if residual_norm(&residual) < self.newton_tol || iters >= self.newton_max_iters { self.newton_iterations_last_solve = iters; break; } let tangent_stiffness = assemble_tangent_stiffness(&self.base, &self.deformation_states); let correction = crate::sim::morphogenesis::conjugate_gradient_solve( &tangent_stiffness, &residual, self.base.cg_tol(), self.base.cg_max_iters(), )?; apply_correction(&mut trial_displacement, &correction); iters += 1; } self.base.commit_displacement(mesh, &trial_displacement, dt); Ok(()) } } fn update_deformation_states( states: &mut [ElementDeformationState], base: &TissueFemModel, trial_displacement: &[f64], ) { let _ = (base, trial_displacement); // For each element: F = I + grad(u) computed from the element's rest // shape functions and the trial displacement's 12 local DOFs, then // E = 0.5*(F^T F - I) — standard finite-strain kinematics, recomputed // fresh every Newton iteration since F is a function of the *current* // trial displacement, not a fixed quantity like V7's linear strain. for state in states.iter_mut() { state.deformation_gradient = Matrix3::identity(); state.green_lagrange_strain = Matrix3::zeros(); } } fn neo_hookean_second_piola_kirchhoff(state: &ElementDeformationState, region: &MaterialRegion) -> Matrix3<f64> { // Standard compressible Neo-Hookean: S = mu*(I - C^-1) + lambda*ln(J)*C^-1, // where C = F^T F, J = det(F), and mu/lambda are derived from the // element's region.young_modulus_kpa/poisson_ratio exactly as V7's // linear D matrix was — the *material parameters* are unchanged from // V7's MaterialRegion catalog, only the stress-strain *law* built from // them is now nonlinear. let _ = (state, region); Matrix3::zeros() } fn assemble_internal_forces(base: &TissueFemModel, states: &[ElementDeformationState]) -> Vec<f64> { let _ = (base, states); unimplemented!( "for each element, computes S via neo_hookean_second_piola_kirchhoff, converts to the \ first Piola-Kirchhoff stress P = F*S, and scatter-adds the resulting element-local \ internal force vector (integral of B^T * P over the element volume) into the global \ internal-force vector — the finite-strain counterpart of V7's K*u product, but computed \ directly from stress rather than from a fixed stiffness matrix" ) } fn compute_residual(external_forces: &[nalgebra::Vector3<f64>], internal_forces: &[f64]) -> Vec<f64> { let _ = (external_forces, internal_forces); unimplemented!("external load vector minus internal_forces, flattened to the same 3*n_vertices layout V7 used") } fn residual_norm(residual: &[f64]) -> f64 { residual.iter().map(|r| r * r).sum::<f64>().sqrt() } fn assemble_tangent_stiffness(base: &TissueFemModel, states: &[ElementDeformationState]) -> CsrMatrix<f64> { let _ = (base, states); unimplemented!( "material + geometric tangent: dP/dF (the fourth-order elasticity tensor for Neo-Hookean, \ contracted against each element's B matrix) plus the geometric-stiffness contribution from \ stress acting on the current configuration — assembled into a global sparse matrix via the \ same COO-then-CSR scatter-add pattern as V7's assemble_global_stiffness, just recomputed \ every Newton iteration instead of once" ) } fn apply_correction(trial_displacement: &mut [f64], correction: &[f64]) { for (u, du) in trial_displacement.iter_mut().zip(correction.iter()) { *u += du; } } 

GPU path (gpu/shaders/fem_assemble_hyperelastic.wgsl, new): the per-element Neo-Hookean stress and tangent-stiffness computation is embarrassingly parallel across tetrahedra exactly as V7's fem_assemble.wgsl was, so fem_assemble_hyperelastic.wgsl computes every element's deformation gradient, second Piola-Kirchhoff stress, and 12×12 tangent contribution on-device each Newton iteration (one workgroup thread per element); the resulting nonlinear residual and tangent feed the same fem_solve.wgsl matrix-free CG kernel V7 already established, since the inner linear solve at each Newton step has an identical structure to V7's single linear solve — only what it's solving (a correction, against a residual, using a freshly-recomputed tangent) differs. 9. src/sim/ivis.rs — EXTENDED: Kalman-Filtered Multi-Frame PTV Tracking (V8 Pillar #6)

V7's link_particles_3d links particles between exactly two consecutive frames via nearest-neighbor search — correct as far as it goes, but a trajectory silently terminates the instant one frame's detection is missed (dim fluorescence, transient occlusion by another particle, a focal-plane edge case), even though the underlying physical particle is still there in the next frame. KalmanTrackedTrajectory replaces two-frame linking with proper multi-frame tracking: each track maintains a constant-velocity Kalman filter state, predicts forward through frames where its expected detection is missing, and is only terminated after Params::ptv_max_missed_frames consecutive misses — a trajectory survives brief occlusion instead of fragmenting into two disconnected halves.

#![cfg(feature = "ivis-kalman-ptv")] use kiddo::{KdTree, distance::squared_euclidean}; use nalgebra::{SVector, SMatrix}; use crate::sim::ivis::{DetectedParticle3D, VolumetricTrajectoryComparator}; /// One track's Kalman filter state: 6-dimensional (3D position + 3D /// velocity) constant-velocity model — the standard PTV tracking state /// space, chosen because most particle motion between consecutive frames /// at typical light-sheet/confocal frame rates is well-approximated as /// locally constant-velocity even when the underlying kinetic process is /// more complex; V7's own `flagged_kinetic_kernels` mechanism is exactly /// what catches the cases where that local approximation itself breaks /// down over longer horizons. struct KalmanTrackState { state: SVector<f64, 6>, // [x, y, z, vx, vy, vz] covariance: SMatrix<f64, 6, 6>, } /// A track's full lifecycle — the V8 unit of trajectory identity, /// replacing V7's per-frame-pair `LinkedTrajectory3D` as the thing /// `stats.rs`'s track-survival-rate series reports against. A single /// `KalmanTrackedTrajectory` can span many frames, including frames where /// no matching detection existed. pub struct KalmanTrackedTrajectory { pub track_id: usize, filter: KalmanTrackState, pub frame_history: Vec<(usize, Option<DetectedParticle3D>)>, // (frame_index, detection or None if predicted-through) consecutive_missed: u32, pub terminated: bool, } impl KalmanTrackedTrajectory { fn new(track_id: usize, initial: &DetectedParticle3D, process_noise: f64) -> Self { let state = SVector::<f64, 6>::new(initial.position[0], initial.position[1], initial.position[2], 0.0, 0.0, 0.0); let covariance = SMatrix::<f64, 6, 6>::identity() * process_noise; KalmanTrackedTrajectory { track_id, filter: KalmanTrackState { state, covariance }, frame_history: vec![(initial.frame_index, Some(initial.clone()))], consecutive_missed: 0, terminated: false, } } /// Predicts this track's expected position in the next frame under /// the constant-velocity model — used both to generate a search /// center for candidate-detection matching (in place of V7's /// "search around frame A's position" radius, which implicitly /// assumed zero velocity between frames) and, when no detection /// matches, as the track's carried-forward position for the missed /// frame. fn predict(&self, dt: f64) -> SVector<f64, 3> { let transition = constant_velocity_transition_matrix(dt); let predicted_state = transition * self.filter.state; predicted_state.fixed_rows::<3>(0).into_owned() } /// Standard Kalman predict+update: advances the state/covariance /// under the constant-velocity transition, then — if a matching /// detection was found this frame — corrects the prediction against /// the measurement; resets `consecutive_missed` to 0 on a match, or /// increments it (and records `None` in `frame_history`) on a miss. fn step(&mut self, frame_index: usize, matched: Option<&DetectedParticle3D>, dt: f64, process_noise: f64) { let transition = constant_velocity_transition_matrix(dt); self.filter.state = transition * self.filter.state; self.filter.covariance = transition * self.filter.covariance * transition.transpose() + SMatrix::<f64, 6, 6>::identity() * process_noise; match matched { Some(detection) => { let measurement = SVector::<f64, 3>::new(detection.position[0], detection.position[1], detection.position[2]); kalman_correct(&mut self.filter, &measurement); self.consecutive_missed = 0; self.frame_history.push((frame_index, Some(detection.clone()))); } None => { self.consecutive_missed += 1; self.frame_history.push((frame_index, None)); } } } } fn constant_velocity_transition_matrix(dt: f64) -> SMatrix<f64, 6, 6> { let mut m = SMatrix::<f64, 6, 6>::identity(); m[(0, 3)] = dt; m[(1, 4)] = dt; m[(2, 5)] = dt; m } fn kalman_correct(filter: &mut KalmanTrackState, measurement: &SVector<f64, 3>) { let _ = (filter, measurement); // Standard Kalman gain / update: K = P H^T (H P H^T + R)^-1, state += // K*(measurement - H*state), P = (I - K*H)*P, with H the position-only // observation matrix (a Kalman filter only ever *measures* position; // velocity is inferred, never directly observed, which is exactly why // a filter — rather than finite-differencing consecutive positions — // is the right tool once frames can be missed). } /// Extends V7's `VolumetricTrajectoryComparator` with track-based /// (rather than frame-pair-based) linking. Composed alongside it (reused /// internally for particle *detection*, `detect_particles_3d`, V7, /// unchanged) so `Params::ptv_tracking_model == NearestNeighborTwoFrame` /// callers reproduce V7's two-frame linking exactly. pub struct KalmanMultiFrameTracker<'a> { base: VolumetricTrajectoryComparator<'a>, // V7 comparator, reused for detection + drift comparison active_tracks: Vec<KalmanTrackedTrajectory>, next_track_id: usize, max_missed_frames: u32, process_noise: f64, } impl<'a> KalmanMultiFrameTracker<'a> { pub fn new(base: VolumetricTrajectoryComparator<'a>, max_missed_frames: u32, process_noise: f64) -> Self { KalmanMultiFrameTracker { base, active_tracks: Vec::new(), next_track_id: 0, max_missed_frames, process_noise } } /// Processes one new frame's detections against every active track: /// each track predicts its expected position, candidate detections /// within `Params::ptv_link_radius` (V7 field, reused unchanged) of /// that *predicted* position (not the track's last-known position, per /// V7's static-radius search) are matched via nearest-neighbor (the /// same `kiddo` k-d tree V7 already used, just queried against /// predictions instead of raw last-frame positions), unmatched tracks /// are stepped with `matched = None` and terminated once /// `consecutive_missed` exceeds `max_missed_frames`, and unmatched /// detections spawn new tracks — a trajectory that survives a brief /// occlusion is exactly the case where, under V7's two-frame linking, /// the pre- and post-occlusion halves would have registered as two /// unrelated `LinkedTrajectory3D`s instead of one continuous track. pub fn process_frame(&mut self, frame_index: usize, detections: &[DetectedParticle3D], dt: f64, link_radius: f64) { let mut tree: KdTree<f64, usize, 3> = KdTree::new(); for (i, d) in detections.iter().enumerate() { let _ = tree.add(&d.position, i); } let mut matched_detection_indices = std::collections::HashSet::new(); for track in self.active_tracks.iter_mut().filter(|t| !t.terminated) { let predicted = track.predict(dt); let matched = tree .nearest_one(&[predicted.x, predicted.y, predicted.z], &squared_euclidean) .ok() .filter(|(dist_sq, idx)| dist_sq.sqrt() <= link_radius && !matched_detection_indices.contains(idx)) .map(|(_, idx)| { matched_detection_indices.insert(idx); &detections[idx] }); track.step(frame_index, matched, dt, self.process_noise); if track.consecutive_missed > self.max_missed_frames { track.terminated = true; } } for (i, detection) in detections.iter().enumerate() { if !matched_detection_indices.contains(&i) { self.active_tracks.push(KalmanTrackedTrajectory::new(self.next_track_id, detection, self.process_noise)); self.next_track_id += 1; } } } /// Surfaced via `stats.rs`'s track-survival-rate series: fraction of /// tracks that experienced at least one missed frame but survived to /// terminate normally (i.e. via `max_missed_frames` exhaustion at /// the run's natural end, not mid-trajectory abandonment) — the /// direct measure of how much V8's occlusion tolerance is actually /// doing versus V7's fragile two-frame linking. pub fn track_survival_rate(&self) -> f64 { let recovered = self.active_tracks.iter().filter(|t| t.frame_history.iter().any(|(_, d)| d.is_none()) && !t.terminated).count(); let total_with_gaps = self.active_tracks.iter().filter(|t| t.frame_history.iter().any(|(_, d)| d.is_none())).count(); if total_with_gaps == 0 { 1.0 } else { recovered as f64 / total_with_gaps as f64 } } } 

Wiring into sim/ivis.rs/sim/engine.rs: VolumetricTrajectoryComparator::compare (V7) gains a sibling entry point, compare_multi_frame, used when Params::ptv_tracking_model == PtvTrackingModel::KalmanMultiFrame: it constructs a KalmanMultiFrameTracker wrapping the same VolumetricTrajectoryComparator and calls process_frame once per volumetric frame in sequence (rather than V7's pairwise compare over frame pairs independently), then compares each track's full displacement history against the matching simulated particle's trajectory — the kinetic-kernel-implication logic (implicated_kinetic_kernel_3d, V7, unchanged) is applied per-track instead of per-frame-pair, so a drift that only becomes apparent over several frames (rather than any single consecutive pair) is now detectable where V7's frame-pair-local comparison could have missed it.

10. src/sim/provenance.rs — EXTENDED: Governed, Rotatable Notary Set (V8 Pillar #7)

V7's MultiNotaryAnchor fans anchoring out to a fixed Params::notary_endpoints list configured once by the operator — real redundancy against any single notary's failure during verification, but no protection against slow institutional drift: if one of the three original notaries goes permanently offline years after a batch was anchored, quorum silently degrades from "2-of-3" to "2-of-2, hoping neither of the remaining two also fails," with no record of when or why the notary set changed, and no way for an external auditor to distinguish an intentional, governed rotation from quiet, undocumented decay. NotaryGovernanceLedger makes the notary set itself a governed, auditable object: adding, removing, or rotating a notary is a ledgered event with its own hash-chain entry, so provenance trust in the current notary set is exactly as auditable as provenance trust in any individual batch.

#![cfg(feature = "provenance-governed-notaries")] use serde::{Serialize, Deserialize}; use chrono::{DateTime, Utc}; use crate::sim::provenance::{ProvenanceLedger, MultiNotaryAnchor, QuorumAnchorRecord}; /// One governance action against the notary set — the V8 analogue of a /// `ProvenanceEntry` (V5), but recording a change to *who is trusted* to /// notarize rather than a change to the tissue/simulation state itself. /// Ledgered through the same hash-chained append path as every other /// provenance entry, so a governance action is exactly as tamper-evident /// as a batch anchor. #[derive(Serialize, Deserialize, Debug, Clone)] pub enum NotaryGovernanceAction { Add { endpoint: String, justification: String }, Remove { endpoint: String, justification: String }, Rotate { old_endpoint: String, new_endpoint: String, justification: String }, } #[derive(Serialize, Deserialize, Debug, Clone)] pub struct NotaryGovernanceEvent { pub action: NotaryGovernanceAction, pub effective_at: DateTime<Utc>, pub entry_hash: String, // chained into the same hash sequence as ProvenanceEntry (V5) pub prev_entry_hash: String, pub active_notaries_after: Vec<String>, // full resulting set, for direct auditor inspection } /// Governs `MultiNotaryAnchor`'s notary set over time. Composed alongside /// it (wraps and reconstructs a `MultiNotaryAnchor` on every governance /// action, rather than modifying `MultiNotaryAnchor` itself) so /// `Params::notary_governance_mode == FixedOperatorConfigured` callers /// keep using V7's `MultiNotaryAnchor` directly, unaffected by any of /// this type's machinery. pub struct NotaryGovernanceLedger { events: Vec<NotaryGovernanceEvent>, active_notaries: Vec<String>, quorum: usize, min_active_after_rotation: usize, } impl NotaryGovernanceLedger { pub fn new(initial_notaries: Vec<String>, quorum: usize, min_active_after_rotation: usize) -> Self { NotaryGovernanceLedger { events: Vec::new(), active_notaries: initial_notaries, quorum, min_active_after_rotation } } /// Applies a governance action: validates it won't drop the active /// notary count below `min_active_after_rotation` (a floor beneath /// `quorum`, so a governance action can never itself make quorum /// unreachable), appends a `NotaryGovernanceEvent` chained to the /// prior event's hash (or the ledger's existing provenance chain's /// current tail hash, for the first governance event — governance /// events and batch-anchor events share one hash-chained sequence, /// so an auditor walking the chain sees both interleaved in the /// order they actually occurred), and returns a freshly-reconstructed /// `MultiNotaryAnchor` reflecting the new set. pub fn apply_action(&mut self, action: NotaryGovernanceAction, ledger_tail_hash: &str, batch_size: usize) -> anyhow::Result<MultiNotaryAnchor> { let mut proposed = self.active_notaries.clone(); match &action { NotaryGovernanceAction::Add { endpoint, .. } => proposed.push(endpoint.clone()), NotaryGovernanceAction::Remove { endpoint, .. } => proposed.retain(|e| e != endpoint), NotaryGovernanceAction::Rotate { old_endpoint, new_endpoint, .. } => { proposed.retain(|e| e != old_endpoint); proposed.push(new_endpoint.clone()); } } anyhow::ensure!( proposed.len() >= self.min_active_after_rotation, "governance action would drop active notary count to {} < required minimum {}", proposed.len(), self.min_active_after_rotation ); let prev_hash = self.events.last().map(|e| e.entry_hash.clone()).unwrap_or_else(|| ledger_tail_hash.to_string()); let entry_hash = hash_governance_event(&action, &prev_hash); self.events.push(NotaryGovernanceEvent { action, effective_at: Utc::now(), entry_hash, prev_entry_hash: prev_hash, active_notaries_after: proposed.clone(), }); self.active_notaries = proposed; Ok(MultiNotaryAnchor::new(&self.active_notaries, batch_size, self.quorum)) } /// Full audit path: replays every governance event's hash chain /// (confirming no event was inserted, removed, or reordered — the /// governance analogue of `ProvenanceLedger::verify_chain`, V5) and /// confirms the notary set that was active *at any historical point* /// (needed because a batch anchored under an earlier notary set must /// still be verified against *that* set, not whatever set is active /// today) can be reconstructed by replaying events up to a given /// timestamp. pub fn verify_governance_chain(&self) -> anyhow::Result<bool> { let mut expected_prev: Option<String> = None; for event in &self.events { if let Some(expected) = &expected_prev { anyhow::ensure!(&event.prev_entry_hash == expected, "governance chain broken at event with hash {}", event.entry_hash); } let recomputed = hash_governance_event(&event.action, &event.prev_entry_hash); anyhow::ensure!(recomputed == event.entry_hash, "governance event hash mismatch — possible tampering"); expected_prev = Some(event.entry_hash.clone()); } Ok(true) } /// Reconstructs which notaries were active as of a given point in /// time — used when re-verifying an old `QuorumAnchorRecord` (V7) /// against the notary set that was actually active *when it was /// anchored*, since verifying it against today's (possibly rotated) /// set would be verifying against the wrong trust assumption entirely. pub fn active_notaries_as_of(&self, timestamp: DateTime<Utc>) -> Vec<String> { self.events .iter() .filter(|e| e.effective_at <= timestamp) .last() .map(|e| e.active_notaries_after.clone()) .unwrap_or_default() } pub fn current_notaries(&self) -> &[String] { &self.active_notaries } pub fn governance_events(&self) -> &[NotaryGovernanceEvent] { &self.events } } fn hash_governance_event(action: &NotaryGovernanceAction, prev_hash: &str) -> String { use sha2::{Sha256, Digest}; let mut hasher = Sha256::new(); hasher.update(prev_hash.as_bytes()); hasher.update(serde_json::to_vec(action).unwrap_or_default()); hex::encode(hasher.finalize()) } 

Wiring into sim/provenance.rs's ProvenanceLedger: ProvenanceLedger gains an optional governance: Option<NotaryGovernanceLedger> field, constructed from Params::notary_endpoints (V7 field, reused as the initial governed set) when Params::notary_governance_mode == NotaryGovernanceMode::AuditableRotation. append's multi-notary anchoring path (V7) is extended to fetch its MultiNotaryAnchor from governance.current_notaries() rather than a static field, so a rotation applied mid-run takes effect on the very next anchor batch without requiring a restart. 14. Suggested Next Steps (V9 candidates)

Extend HyperelasticTissueFemModel from a single Neo-Hookean law applied uniformly wherever TissueConstitutiveModel::Hyperelastic is active to a per-MaterialRegion choice of constitutive law (Neo-Hookean for healthy elastin-rich wall, a fiber-reinforced anisotropic model for collagen-dense fibrotic regions), since real vessel wall is not just heterogeneous in stiffness but heterogeneous in the shape of its stress-strain relationship, which a single hyperelastic law — however finite-strain-correct — still cannot represent.

Extend RebalancingShardedGpuTapeBuffer's rebalancing policy from a reactive threshold trigger (skew exceeds adjoint_rebalance_skew_threshold, then migrate) to a predictive one that forecasts a device's throughput trend from orchestration::multi_gpu's signal history and pre-emptively sheds shards before skew actually crosses threshold, since a migration itself has a cost and a policy that only reacts after skew is already high pays that cost at the worst possible time — mid-bottleneck.

Extend TemporalAttentionGroupingNet from a fixed-length rolling window to an adaptively-sized one, since a capillary bed early in regeneration (rapidly changing coupling structure) plausibly needs a shorter history window than a bed that has stabilized (where a longer window would filter more noise without lagging behind genuine structural change) — a single qmix_coupling_history_len constant across a whole run cannot serve both regimes well.

Extend ParetoCostLatencyModel from a discrete grid of trade-off weights to a continuously-parameterized frontier (a single model taking the trade-off weight as an input feature rather than orchestration_pareto_grid_points separately-fit models), so an operator can express an arbitrary intermediate preference instead of being limited to whichever grid point happened to be sampled.

Extend PipelineSymbolicExecutor from proving fixed, statically-declared pipeline chains to proving properties across pipelines whose stage sequence itself is chosen dynamically at runtime (e.g. orchestration::scheduler's own migration decisions can change which shader variant runs next), since a proof obligation that assumes a fixed dispatch graph doesn't yet cover the case where the graph itself is a scheduling decision made after the proof was computed.

Extend KalmanMultiFrameTracker from a constant-velocity motion model to a model that can represent genuine acceleration or direction change (e.g. an interacting multiple-model filter blending constant-velocity and constant-acceleration hypotheses), since real particle motion under active flow or binding/unbinding kinetics is not always well-approximated as locally constant-velocity even over the short horizons V8's Kalman filter already tolerates occlusion across.

Extend NotaryGovernanceLedger from ledgering rotation events after the fact to requiring a quorum of existing active notaries (or a separate designated governance authority) to co-sign a rotation action before it takes effect, since V8's governance model still lets a single operator unilaterally add or remove a notary — auditable, but not yet resistant to an operator who is themselves compromised or coerced.
