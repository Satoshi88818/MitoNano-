The Unification: One Engine, Everything Composed

Every piece built so far — motion, PK decay, field diffusion, agent→field release, field→agent uptake, agent-agent exchange via the spatial index — now runs together in a single tick loop, and is proven to compose correctly, not just proven correct in isolation.

Status: compiles clean, zero warnings. 3 new tests, all passing. Full crate: 50/50 tests passing in ~1.75s.

Why this is the right kind of "last" test

Every previous module was validated on its own. That leaves exactly the failure mode integration testing exists to catch: two individually-correct pieces that misbehave only when combined — an ordering assumption that only breaks under load, a shared piece of state one module mutates in a way another didn't expect. UnifiedEngine runs the whole stack together and re-derives the same two kinds of invariants used throughout this crate — exact conservation and exact bookkeeping identities — now under maximum simultaneous complexity, plus a full-system rerun of the brute-force cross-validation from last time.

Design: one fixed, documented tick order

Per tick, per agent (sequentially): move → sense the field and withdraw an uptake amount (capped, can't go negative) → gain that into payload → exact decay, split between release and elimination → deposit released mass into the field. Once every agent has done that: rebuild the spatial index and run pairwise agent-agent exchange. Finally, the field diffuses one step.

This ordering is a choice, not the only physically defensible one — but it's fixed and documented, which is what makes it testable at all.

The tests

full_system_exactly_conserves_mass_with_zero_elimination — with k_elim=0, all three transfer pathways active (release, uptake, agent-agent exchange) every tick, total combined mass (agents + field) stays exactly constant to 1e-9 across 400 ticks, 40 agents.

full_system_matches_exact_bookkeeping_identity_under_maximum_complexity — motion, diffusion, release, uptake, elimination, and agent-agent exchange all active simultaneously. No closed form exists anymore, but total_combined_mass(t) == C0 − cumulative_eliminated(t) must hold exactly, since every pathway except elimination is a self-canceling transfer. Held to 1e-9 across 500 ticks, 50 agents — the single most demanding correctness check in the crate.

spatial_index_matches_brute_force_with_full_feedback_active — the real integration-bug hunt. Two full unified simulations from identical initial conditions and identical RNG-driven motion, differing only in whether agent-agent exchange uses the spatial index or brute-force search, while release/uptake/elimination/diffusion are simultaneously modifying every agent's payload every tick. Payload and field-mass trajectories matched to <1e-9 at every tick across 200 ticks, 45 agents. This is exactly the kind of subsystem-interaction bug that testing the index alone (orchestration.rs) or the interaction loop alone (interaction.rs) couldn't have caught — it only shows up when both are running under real feedback pressure at once. It didn't show up.
