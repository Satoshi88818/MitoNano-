MitoNano Engine V9 — From Adaptive, History-Aware, Governed Clinical Infrastructure to Heterogeneous-Law, Predictive, Continuously-Parameterized, Dynamically-Proven, Multi-Model, Quorum-Governed Clinical Infrastructure

This document integrates the seven "Suggested Next Steps (V9 candidates)" from MitoNanoV8.txt §14 into the V8 architecture, producing a coherent V9 Rust workspace. As with every prior major version, all V1–V8 modules are preserved and extended, not replaced — existing params.json files keep deserializing (new fields carry #[serde(default)]), existing CLI invocations keep working, and every new subsystem is additive and feature-gated behind a new Cargo feature that layers on top of its V8 parent feature rather than replacing it.

Where V8's theme was adaptivity, memory, and governance, V9's theme is heterogeneity-within-heterogeneity, foresight, and honesty about what a fixed proof or fixed model cannot yet see: a single hyperelastic law is no longer applied uniformly across a tissue region, a rebalancing policy no longer waits for skew to actually occur, a coupling-history window is no longer one constant for a whole run, a cost/latency trade-off is no longer sampled at only a handful of grid points, a pipeline proof is no longer restricted to dispatch graphs fixed before the proof was computed, a particle's motion is no longer assumed locally constant-velocity, and a notary rotation is no longer an action one operator can take unilaterally.

4. src/sim/morphogenesis.rs — EXTENDED: Per-Region Anisotropic Hyperelasticity (V9 Pillar #1)

V8's HyperelasticTissueFemModel applies one Neo-Hookean law uniformly to every element regardless of MaterialRegion — correct for elastin-rich healthy wall, but collagen-dense fibrotic regions genuinely resist stretch anisotropically, stiffening much faster along the dominant collagen-fiber direction than transverse to it; representing that with an isotropic law, however finite-strain-correct, forces a single stress-strain shape onto tissue that does not have one shape. AnisotropicHyperelasticModel keeps V8's Neo-Hookean branch as the default per-element law and adds a fiber-reinforced branch, selected per element from its owning MaterialRegion's tag, so a single mesh can contain both laws simultaneously.

#![cfg(feature = "fem-anisotropic")] use crate::sim::morphogenesis::{ HyperelasticTissueFemModel, ElementDeformationState, MaterialRegion, TissueFemModel, }; use nalgebra::{Matrix3, Vector3}; /// Which constitutive law an individual `MaterialRegion` uses — V8's /// `MaterialRegion` carried only mechanical *parameters* (Young's /// modulus, Poisson ratio); V9 additionally tags each region with which /// *law* those parameters feed. `Isotropic` regions are unaffected by /// this pillar at all — they still route through V8's /// `neo_hookean_second_piola_kirchhoff` exactly. #[derive(Clone, Copy, PartialEq, Eq, Debug, Default)] pub enum RegionConstitutiveLaw { #[default] Isotropic, // V8: plain Neo-Hookean FiberReinforced, // V9: Neo-Hookean matrix term + anisotropic fiber term } /// A region's dominant collagen-fiber direction and dispersion — the /// structure-tensor inputs a Holzapfel-Gasser-Ogden-style anisotropic /// term needs beyond what V8's `MaterialRegion` already carried /// (Young's modulus / Poisson ratio alone cannot express a *direction*). #[derive(Clone, Copy, Debug)] pub struct FiberStructureTensor { pub mean_direction: Vector3<f64>, // unit vector, rest-configuration frame pub dispersion_kappa: f64, // 0 = fully aligned, ->1/3 = isotropic dispersion pub fiber_stiffness_kpa: f64, // additional anisotropic stiffness scale } /// Extends V8's `HyperelasticTissueFemModel` with a per-element law /// dispatch. Composed alongside it (wraps `base`, V8's model, reused /// unchanged for its Newton-Raphson loop structure and its Neo-Hookean /// branch) so `Params::tissue_constitutive_model == Hyperelastic` /// callers reproduce V8's uniform-law behavior exactly — this type's /// `assemble_and_solve_anisotropic` only differs from V8's /// `assemble_and_solve_hyperelastic` in which per-element stress /// function `assemble_internal_forces`/`assemble_tangent_stiffness` /// dispatch to, not in the outer Newton loop itself. pub struct AnisotropicHyperelasticModel { base: HyperelasticTissueFemModel, // V8 model, reused for Newton loop + isotropic branch region_laws: Vec<RegionConstitutiveLaw>, // one per MaterialRegion fiber_tensors: Vec<Option<FiberStructureTensor>>, // Some(..) iff region_laws[i] == FiberReinforced } impl AnisotropicHyperelasticModel { pub fn from_hyperelastic_model( base: HyperelasticTissueFemModel, region_laws: Vec<RegionConstitutiveLaw>, fiber_tensors: Vec<Option<FiberStructureTensor>>, ) -> Self { AnisotropicHyperelasticModel { base, region_laws, fiber_tensors } } /// Identical outer Newton-Raphson structure to V8's /// `assemble_and_solve_hyperelastic` — the loop body calls /// `assemble_internal_forces_regional`/`assemble_tangent_stiffness_regional` /// in place of V8's uniform-law versions, and those functions /// dispatch *per element* on `self.region_laws[element.region_id]` /// rather than assuming one law for the whole mesh. pub fn assemble_and_solve_anisotropic( &mut self, mesh: &mut crate::sim::morphogenesis::LagrangianVesselMesh, external_forces: &[Vector3<f64>], dt: f64, ) -> anyhow::Result<()> { self.base.assemble_and_solve_hyperelastic_with_stress_fn( mesh, external_forces, dt, |state, region, region_id| self.regional_second_piola_kirchhoff(state, region, region_id), ) } /// Per-element law dispatch: `Isotropic` regions call V8's /// `neo_hookean_second_piola_kirchhoff` unmodified; `FiberReinforced` /// regions add an anisotropic term built from that element's /// `FiberStructureTensor` on top of the same isotropic matrix term — /// the matrix (ground-substance) contribution is never dropped, only /// supplemented, since real fibrotic tissue still has a non-fiber /// matrix surrounding the collagen network. fn regional_second_piola_kirchhoff( &self, state: &ElementDeformationState, region: &MaterialRegion, region_id: usize, ) -> Matrix3<f64> { let matrix_term = crate::sim::morphogenesis::neo_hookean_second_piola_kirchhoff(state, region); match self.region_laws[region_id] { RegionConstitutiveLaw::Isotropic => matrix_term, RegionConstitutiveLaw::FiberReinforced => { let tensor = self.fiber_tensors[region_id] .expect("FiberReinforced region must carry a FiberStructureTensor"); matrix_term + anisotropic_fiber_stress(state, &tensor) } } } } /// Holzapfel-Gasser-Ogden-style anisotropic term: contracts the current /// Green-Lagrange strain against the fiber structure tensor /// `kappa*I + (1-3*kappa)*(a0 (x) a0)` (the dispersion-weighted blend of /// isotropic and fully-aligned fiber contributions), scaled by /// `fiber_stiffness_kpa`, and — critically — only activated in tension /// along the fiber direction (collagen fibers resist stretch, not /// compression, so a naive symmetric term would over-stiffen a /// compressed fiber bundle in a way real collagen does not). fn anisotropic_fiber_stress(state: &ElementDeformationState, tensor: &FiberStructureTensor) -> Matrix3<f64> { let _ = (state, tensor); unimplemented!( "computes the structure tensor H = kappa*I + (1-3*kappa)*(a0 outer a0), the fiber \ invariant I4 = a0 . (C a0) via state.green_lagrange_strain's implied C = 2E + I, \ gates the term to zero when I4 <= 1.0 (fiber in compression or unstretched), and \ returns fiber_stiffness_kpa * (I4 - 1.0) * H as the anisotropic Second \ Piola-Kirchhoff contribution — same tensor algebra V8's isotropic term already used, \ contracted against a direction-carrying tensor instead of the identity" ) } 

GPU path (gpu/shaders/fem_assemble_anisotropic.wgsl, new): the per-element dispatch is embarrassingly parallel exactly as V8's fem_assemble_hyperelastic.wgsl was, so fem_assemble_anisotropic.wgsl adds a per-element region_law: u32 storage buffer (populated once at mesh load, unchanged across Newton iterations within a tick) that each workgroup thread branches on before computing its element's stress and tangent contribution — Isotropic threads execute V8's existing code path byte-for-byte, FiberReinforced threads additionally read that element's fiber_direction/fiber_kappa/fiber_stiffness from a second small per-region uniform buffer. Both shaders remain routed through gpu/shader_verifier.rs before dispatch.

Wiring into sim/morphogenesis.rs/sim/engine.rs: LagrangianVesselMesh's hyperelastic path gains a third sibling: when Params::tissue_constitutive_model == TissueConstitutiveModel::AnisotropicRegional, the mesh constructs Option<AnisotropicHyperelasticModel> (wrapping the same HyperelasticTissueFemModel internally, itself wrapping the same V7 TissueFemModel — three layers deep, each byte-identical to its parent in its own default mode). Simulation::step branches at the same call site V6–V8 established: LinearSmallStrain and Hyperelastic are unchanged from V8; AnisotropicRegional calls assemble_and_solve_anisotropic() instead. MaterialRegion loading (V7, unchanged) is extended so a region's config may optionally carry a constitutive_law tag and fiber_structure block — a region omitting both defaults to RegionConstitutiveLaw::Isotropic, so existing region catalogs keep loading and keep behaving as V8's uniform Neo-Hookean would.

5. src/sim/adjoint.rs / src/gpu/adjoint_gpu.rs — EXTENDED: Predictive Shard Rebalancing (V9 Pillar #2)

V8's RebalancingShardedGpuTapeBuffer only migrates once occupancy skew has already crossed adjoint_rebalance_skew_threshold — a real improvement over V7's static partition, but a migration itself costs a device-to-device transfer, and paying that cost only once the backward sweep is already bottlenecked means the transfer's own latency stacks directly onto the very bottleneck it's trying to relieve. V8's KalmanMultiFrameTracker gives every track a single constant-velocity filter — a real improvement over V7's two-frame linking, and adequate for particle motion that is locally smooth, but real particle motion under active flow or binding/unbinding kinetics genuinely accelerates or changes direction, and a pure constant-velocity filter systematically lags behind (or overshoots past) a maneuvering particle's true position, which can itself cause a correct match to fall outside ptv_link_radius and be lost. ImmMultiFrameTracker blends a constant-velocity hypothesis and a constant-acceleration hypothesis per track, weighting each by a model probability that updates every frame based on which hypothesis is currently predicting better — recovering V8's exact CV-only behavior when a track's dynamics never call for anything else (the CA hypothesis's model probability simply stays near zero).

/// (2) runs each sub-filter's own predict+update against the matched /// detection (or predicts-through on a miss, as V8's `step` already /// did for its single filter); (3) updates each model's probability /// from its own prediction's residual likelihood — a sub-filter whose /// recent predictions matched actual detections better gets a higher /// probability going into the next cycle; (4) combines both /// sub-filters' post-update states into the single reported /// `combined_state` via the same probability weighting `predict_combined` /// used. fn step(&mut self, frame_index: usize, matched: Option<&DetectedParticle3D>, dt: f64, process_noise: f64) { mix_states(&mut self.cv_track, &mut self.ca_state, self.model_probability_cv, self.model_probability_ca, self.switch_probability); self.cv_track.step(frame_index, matched, dt, process_noise); // V8, unchanged sub-step constant_accel_step(&mut self.ca_state, matched, dt, process_noise); let (p_cv, p_ca) = update_model_probabilities( self.model_probability_cv, self.model_probability_ca, &self.cv_track, &self.ca_state, matched, ); self.frame_history.push((frame_index, None)); } } } } fn constant_accel_predict_position(state: &ConstantAccelState, dt: f64) -> SVector<f64, 3> { let _ = (state, dt); unimplemented!("standard constant-acceleration kinematics: x' = x + v*dt + 0.5*a*dt^2, applied per axis") } fn constant_accel_step(state: &mut ConstantAccelState, matched: Option<&DetectedParticle3D>, dt: f64, process_noise: f64) { let _ = (state, matched, dt, process_noise); self.next_track_id += 1; } } } /// Surfaced via `stats.rs`'s IMM model-probability series: the /// fraction of currently-active tracks whose CA model probability /// exceeds 0.5 — a direct measure of how much genuinely maneuvering /// (non-constant-velocity) motion this run's dataset actually /// contains, independent of `track_survival_rate` (V8, still /// reported unchanged, since occlusion tolerance and maneuver /// tolerance are different failure modes IMM and V8's plain Kalman /// filter both still separately address). pub fn fraction_tracks_currently_maneuvering(&self) -> f64 { let maneuvering = self.active_tracks.iter().filter(|t| !t.terminated && t.model_probability_ca > 0.5).count(); let active = self.active_tracks.iter().filter(|t| !t.terminated).count(); if active == 0 { 0.0 } else { maneuvering as f64 / active as f64 } } }

10. src/sim/provenance.rs — EXTENDED: Quorum-Cosigned Notary Governance (V9 Pillar #7)

V8's NotaryGovernanceLedger makes notary-set changes auditable — a real improvement over V7's silent, undocumented drift — but apply_action still lets a single operator unilaterally add, remove, or rotate a notary; the action is recorded honestly, but nothing stops it from being taken unilaterally in the first place, which means an operator who is themselves compromised or coerced can still rotate out every independent notary and rotate in ones they control, and the resulting ledger entry looks exactly as valid as a legitimate rotation. QuorumGovernedNotaryLedger requires a quorum of the currently active notaries (or a separately designated governance authority) to cryptographically co-sign a proposed action before it takes effect — the proposal itself is still ledgered immediately (so a rejected or expired proposal is exactly as auditable as an accepted one), but active_notaries only actually changes once notary_governance_cosign_quorum valid co-signatures are collected within notary_governance_cosign_window_secs.

14. Suggested Next Steps (V10 candidates)

Extend AnisotropicHyperelasticModel from a fixed per-region fiber direction (set once at mesh load) to one that co-evolves with tissue remodeling — real collagen fiber orientation itself reorganizes in response to sustained stress direction over a regeneration or fibrotic-progression timescale, and a FiberStructureTensor frozen at its rest-configuration value cannot represent a vessel wall whose fiber architecture is itself adapting to the loads V9's finite-strain solve is computing.

Extend PredictiveShardedGpuTapeBuffer's single-device linear trend forecaster to a forecaster that also accounts for cross-device correlation — a co-scheduled job stealing cycles or a shared thermal envelope throttling several devices at once produces correlated degradation the current per-device-independent forecast cannot anticipate, so a migration plan optimized against independent forecasts can shed load onto a device that is itself about to degrade for the same underlying reason.

Extend AdaptiveTemporalAttentionGroupingNet's window-length controller from reacting to recent coupling-score variance to anticipating a volatility regime change before it happens — much as V9 Pillar #2 made shard rebalancing predictive rather than reactive, the window-sizing controller itself still only ever reacts to volatility already observed, one reclustering cadence behind a genuine regime shift.

Extend ContinuousParetoFrontierModel from a single global frontier fit to one conditioned on live system state (current queue depth, thermal headroom, time-of-day load pattern) — an operator's operator_preferred_tradeoff_weight currently maps to the same predicted frontier point regardless of whether the system is under light or heavy contention, when the actually-achievable frontier itself shifts with system state.

Extend DynamicPipelineSymbolicExecutor from proving handoff safety across a max_branch_fanout-bounded set of scheduler-selectable next stages to also proving a liveness property — that the scheduler's own dispatch-graph selection policy is guaranteed to eventually choose a branch with a valid proof, rather than only proving each individually-reachable branch safe if chosen, since a policy that could in principle select an unproven branch beyond max_branch_fanout is not yet ruled out by V9's safety-only proof obligation.

Extend ImmMultiFrameTracker from a fixed two-hypothesis (CV/CA) model set to a data-driven model set — real particle-binding/unbinding kinetics can produce motion signatures (e.g. sudden near-stop, not smooth deceleration) that neither constant-velocity nor constant-acceleration represents well, and a third or fourth hypothesis tuned from sim::ivis's own historical kinetic-kernel catalog would let IMM's model probabilities express "this looks like binding," not just "this looks accelerating."

Extend QuorumGovernedNotaryLedger from requiring a quorum of currently-active notaries to co-sign a rotation to also supporting a designated external governance authority whose co-signature is required in addition to (not merely as an alternative to) notary quorum for the specific case of removing a notary below notary_min_active_after_rotation's configured floor — V9's quorum protects against a single compromised operator, but not yet against a majority of the active notary set itself being coerced or compromised simultaneously, which a designated authority external to the notary set entirely (e.g. a separate institutional review board) is better positioned to catch.
