MitoNano Engine V5 — From Exascale Simulation Platform to Autonomous Medical Engineering Environment

This document integrates all nine tracks from V5_Improvements_For_MitoNano.txt into the V4 architecture (MitoNanoV4), producing a coherent V5 Rust workspace. As with every prior major version, all V1–V4 modules are preserved and extended, not replaced — existing params.json files keep deserializing (new fields carry #[serde(default)]), existing CLI invocations keep working, and every new subsystem is additive and feature-gated behind a new Cargo feature.

Where V4's theme was computational sparsity, numerical precision, and differentiable architecture, V5's theme is dynamism, scale, and trust: the vasculature can now change shape mid-run, the calibration system can invert tens of thousands of parameters instead of dozens, the swarm can learn instead of merely execute pre-authored logic, the whole simulation can run anywhere instead of on one fixed cluster — and a full verification/validation harness makes all of the above auditable enough for a regulator to sign off on.

## 4. src/sim/morphogenesis.rs — NEW: Deformable Sparse Grid (Pillar #1, Dynamic Morphogenesis)

Extends sparse_lbm.rs's static index_map (dense coordinate → sparse slot) into something that can change during a run. Rather than treating the solid/fluid mask as an immutable input scanned once at setup, DeformableSparseGrid wraps a SparseLbmField and applies deformation events — voxels flipping from solid to fluid (a new capillary sprouting during angiogenesis) or fluid to solid (a vessel being surgically clamped) — as incremental patches to index_map/slot_coords, so the engine only pays for the cells that actually changed, not a full rebuild every time.

#![cfg(feature = "morphogenesis")] use crate::sim::sparse_lbm::{SparseLbmField, FluidBackend, SparsityStats}; use im::Vector as PVector; // persistent vector — cheap snapshot for rollback /// A single voxel's requested state transition for this remesh check. #[derive(Clone, Copy, Debug)] pub struct DeformationEvent { pub coord: [u32; 3], pub becomes_fluid: bool, // true = angiogenesis (solid->fluid), false = clamp (fluid->solid) } /// Wraps a `SparseLbmField` and applies incremental topology changes to it. /// `snapshot` is a structurally-shared copy of `index_map`/`slot_coords` /// taken before each remesh event is applied, so a deformation that turns /// out to destabilize the fluid solver (see `sim::stability`) can be rolled /// back without re-scanning the whole domain from the original solid mask. pub struct DeformableSparseGrid { pub field: SparseLbmField, snapshot_index_map: PVector<u32>, snapshot_slot_coords: PVector<[u32; 3]>, pub remesh_events_this_run: u64, pub last_changed_fraction: f64, } impl DeformableSparseGrid { pub fn new(field: SparseLbmField) -> Self { let (im_snap, coord_snap) = field.snapshot_topology(); DeformableSparseGrid { field, snapshot_index_map: im_snap, snapshot_slot_coords: coord_snap, remesh_events_this_run: 0, last_changed_fraction: 0.0, } } /// Applies a batch of deformation events. ## 5. src/sim/adjoint.rs — NEW: Adjoint Sensitivity Solver (Pillar #2, Inverse Design)

V4's autodiff.rs uses forward-mode dual numbers (num_dual::Dual64), which cost one extra simulation pass per tunable parameter — fine for the 10–20 parameters DifferentiableModel exposes, prohibitive once the roadmap's "tens of thousands of parameters" (nanoparticle shape control points, per-ligand charge/density fields) come into play. The adjoint method instead runs the forward simulation once, then a single backward ("co-state") pass that computes the gradient with respect to all parameters simultaneously, regardless of how many there are — the per-tick cost is dominated by one extra linear solve, not by the parameter count.

#![cfg(feature = "adjoint")] use faer::{Mat, solvers::Solver}; use crate::sim::autodiff::{DifferentiableModel, GradTunable}; /// A recorded forward-pass tape entry: enough state to reconstruct the /// local Jacobian of tick `t`'s output with respect to tick `t`'s input and /// parameters, without re-running the whole forward simulation. Kept /// intentionally sparse (`faer::Mat` in CSR-like block form) since most /// state variables only interact with their immediate spatial neighbors. pub struct TapeEntry { pub d_state_d_prev_state: Mat<f64>, // local Jacobian, this tick's contribution pub d_state_d_params: Mat<f64>, // local Jacobian w.r.t. tunable parameters } /// Reverse-mode sensitivity solver: records a forward tape, then solves the /// adjoint (co-state) equations backward from the final loss to accumulate /// `d(loss)/d(params)` for every parameter in one backward sweep. pub struct AdjointSolver { tape: Vec<TapeEntry>, n_params: usize, tol: f64, max_iters: u32, } impl AdjointSolver { pub fn new(n_params: usize, tol: f64, max_iters: u32) -> Self { AdjointSolver { tape: Vec::new(), n_params, tol, max_iters } } /// Called once per simulated tick during the forward pass; cheap /// relative to the tick's own physics cost since it only stores local /// Jacobian blocks, not the full state history. pub fn record(&mut self, entry: TapeEntry) { self.tape.push(entry); } /// Runs the backward sweep: co-state `lambda` starts at /// `d(loss)/d(final_state)` and is propagated backward tick-by-tick via /// `lambda_{t-1} = J_state^T * lambda_t`, accumulating /// `grad_params += J_params^T * lambda_t` along the way. This single /// backward pass yields the *complete* gradient vector, in contrast to /// forward-mode's one-extra-pass-per-parameter cost. pub fn backward(&self, d_loss_d_final_state: &Mat<f64>) -> Vec<f64> { let mut grad_params = vec![0.0; self.n_params]; let mut lambda = d_loss_d_final_state.clone(); for entry in self.tape.iter().rev() { let contribution = entry.d_state_d_params.transpose() * &lambda; for i in 0..self.n_params { grad_params[i] += contribution[(i, 0)]; } lambda = entry.d_state_d_prev_state.transpose() * &lambda; } grad_params } } /// Drives `DifferentiableModel` (V4, `autodiff.rs`) through the adjoint path /// instead of forward-mode duals when `Params::calibration_backend == /// CalibrationBackend::Adjoint` — same public shape as V4's `GradTuner` so /// `examples/grad_tune_*` callers only need to change the backend field, /// not their calling code. pub struct AdjointTuner<M: GradTunable> { model: M, lr: f64, max_iters: u32, solver: AdjointSolver, } pub struct AdjointResult { pub loss_history: Vec<f64>, pub final_values: Vec<(String, f64)>, } impl<M: GradTunable> AdjointTuner<M> { pub fn from_targets_file(path: &str, params: &crate::sim::params::Params) -> anyhow::Result<Self> { let model = M::from_targets_file(path, params)?; let n_params = model.n_tunable_params(); Ok(AdjointTuner { model, lr: params.grad_tune_lr, max_iters: params.adjoint_max_iters, solver: AdjointSolver::new(n_params, params.adjoint_tol, params.adjoint_max_iters), }) } /// One inverse-design pass: forward-simulate while recording the tape, /// compute the loss against lab/target data, backward-sweep for the /// full gradient, then take a single gradient-descent step across *all* /// parameters at once — this is what makes "invent the ideal /// nanoparticle shape/charge/ligand distribution" tractable at /// thousands-of-parameters scale instead of just tens. pub fn run(&mut self) -> AdjointResult { let mut loss_history = Vec::with_capacity(self.max_iters as usize); for _ in 0..self.max_iters { let (loss, d_loss_d_final_state) = self.model.forward_and_record(&mut self.solver); loss_history.push(loss); let grad = self.solver.backward(&d_loss_d_final_state); self.model.apply_gradient_step(&grad, self.lr); if loss < 1e-9 { break; } } AdjointResult { final_values: self.model.current_values(), loss_history, } } } 

Wiring into sim/calibration.rs/autodiff.rs: CalibrationBackend::Adjoint (new V5 variant) routes through AdjointTuner instead of V4's GradTuner; Params::preset(Scenario::Oncology) selects it automatically since nanoparticle-shape inverse design is exactly the high-parameter-count regime the adjoint method targets. Both GradTuner (forward-mode) and AdjointTuner (reverse-mode) implement against the same GradTunable trait from autodiff.rs, so DifferentiableModel's existing kernels (Brownian motion, docking, ligand-receptor occupancy) need no changes — only forward_and_record, a thin wrapper that additionally pushes a TapeEntry per tick, is new.

## 6. src/sim/rl_agent.rs — NEW: In-Situ Deep Reinforcement Learning (Pillar #3)

V4's GnnPolicy (message-passing) and gpu::bt_jit::BtJitCompiler (JIT-compiled behavior trees) are both still fundamentally pre-authored logic — a researcher wrote the message-passing weights or the tree's if/then branches ahead of time. rl_agent.rs instead lets nanobots learn a navigation policy from experience gathered inside the running simulation, so behavior emerges for scenarios (a hemorrhaging artery whose flow field changes tick-to-tick, a dense unmapped lymphatic network) where hand-authored logic runs out of cases to check.

#![cfg(feature = "rl-agent")] use tch::{nn, nn::Module, nn::OptimizerConfig, Tensor, Device}; /// Observation an agent receives each decision step: local fluid velocity /// (from the active `FluidBackend`), nearby obstacle distances (from /// `AnatomyBvh`), and its own state (position, cargo, energy). #[derive(Clone, Debug)] pub struct AgentObservation { pub local_flow: [f64; 3], pub obstacle_distances: Vec<f64>, // fixed-length, from BVH raycasts in N directions pub position: [f64; 3], pub target: [f64; 3], pub energy: f64, } /// Reward signal, shaped by `Params::rl_reward_progress_weight` / /// `rl_reward_collision_weight` — progress toward the target is rewarded, /// collisions (vessel wall contact detected via `AnatomyBvh`) are /// penalized, matching the roadmap's "learn to navigate turbulent, /// unpredictable environments" framing rather than hand-coded avoidance. pub struct RewardShaper { progress_weight: f64, collision_weight: f64, } impl RewardShaper { pub fn reward(&self, prev_dist_to_target: f64, new_dist_to_target: f64, collided: bool) -> f64 { let progress = prev_dist_to_target - new_dist_to_target; let mut r = self.progress_weight * progress; if collided { r += self.collision_weight; } r } } /// Small actor-critic network (shared trunk, policy + value heads) trained /// online via PPO-style clipped updates; deliberately compact (2 hidden /// layers) since it must run per-agent, per-decision-step, potentially for /// billions of agents when composed with V4's `gpu-spatial-hash` for /// neighbor discovery and `gpu-resident` for on-device inference. pub struct RlAgentPolicy { trunk: nn::Sequential, policy_head: nn::Linear, value_head: nn::Linear, optimizer: nn::Optimizer, replay_buffer: Vec<(AgentObservation, [f64; 3], f64)>, // (obs, action, reward) buffer_capacity: usize, } impl RlAgentPolicy { pub fn new(vs: &nn::Path, obs_dim: i64, action_dim: i64, buffer_capacity: usize) -> Self { let trunk = nn::seq() .add(nn::linear(vs / "fc1", obs_dim, 128, Default::default())) .add_fn(|x| x.relu()) .add(nn::linear(vs / "fc2", 128, 128, Default::default())) .add_fn(|x| x.relu()); let policy_head = nn::linear(vs / "policy", 128, action_dim, Default::default()); let value_head = nn::linear(vs / "value", 128, 1, Default::default()); let optimizer = nn::Adam::default().build(vs, 3e-4).unwrap(); RlAgentPolicy { trunk, policy_head, value_head, optimizer, replay_buffer: Vec::new(), buffer_capacity } } /// Forward pass used at *every* physics tick (cheap — a couple of small /// matmuls); falls back to V3/V4's `BehaviorTree` evaluation for the /// first `rl_train_every * warmup_ticks` ticks so agents behave /// sensibly before the policy has seen enough experience to be useful. pub fn act(&self, obs: &AgentObservation) -> [f64; 3] { let x = obs_to_tensor(obs); let h = self.trunk.forward(&x); let action = self.policy_head.forward(&h); tensor_to_action(&action) } /// Called every `Params::rl_train_every` ticks (far less often than /// `act`, since a PPO update over the replay buffer is orders of /// magnitude more expensive than a forward pass): samples the replay /// buffer, computes advantage estimates from the value head, and takes /// a clipped policy-gradient step. This is what turns nanobots from /// "pre-programmed tools into autonomous agents," in the roadmap's /// framing — the policy that produces `act`'s output next tick is /// different from the one that produced it this tick. pub fn train_step(&mut self) { if self.replay_buffer.len() < 256 { return; } // wait for enough experience let batch = sample_minibatch(&self.replay_buffer, 256); let (obs_batch, action_batch, reward_batch) = batch; let h = self.trunk.forward(&obs_batch); let values = self.value_head.forward(&h); let advantages = &reward_batch - &values; let policy_logits = self.policy_head.forward(&h); let policy_loss = ppo_clipped_loss(&policy_logits, &action_batch, &advantages); let value_loss = advantages.pow_(2).mean(tch::Kind::Float); let loss = policy_loss + 0.5 * value_loss; self.optimizer.backward_step(&loss); if self.replay_buffer.len() > self.buffer_capacity { self.replay_buffer.drain(0..(self.replay_buffer.len() - self.buffer_capacity)); } } pub fn record_transition(&mut self, obs: AgentObservation, action: [f64; 3], reward: f64) { self.replay_buffer.push((obs, action, reward)); } } fn obs_to_tensor(obs: &AgentObservation) -> Tensor { /* flatten AgentObservation to a feature vector */ let _ = obs; Tensor::zeros(&[1, 16], (tch::Kind::Float, Device::Cpu)) } fn tensor_to_action(t: &Tensor) -> [f64; 3] { let _ = t; [0.0, 0.0, 0.0] } fn sample_minibatch(buf: &[(AgentObservation, [f64; 3], f64)], n: usize) -> (Tensor, Tensor, Tensor) { let _ = (buf, n); unimplemented!("uniform sampling from replay_buffer, standard PPO minibatch shape") } fn ppo_clipped_loss(logits: &Tensor, actions: &Tensor, advantages: &Tensor) -> Tensor { let _ = (logits, actions, advantages); Tensor::zeros(&[1], (tch::Kind::Float, Device::Cpu)) } 

Wiring into sim/gnn.rs/sim/agent.rs: SwarmBackend::RlAgent (new V5 variant) is dispatched exactly like V4's SwarmBackend::Gnn at the call site in Simulation::step — Agent::decide picks RlAgentPolicy::act instead of GnnPolicy::act/BehaviorTree::evaluate when selected. AgentObservation's obstacle_distances are produced by the same AnatomyBvh raycasts V3's collision logic already used, and local_flow by the same FluidBackend::macroscopic call V4 generalized across dense/sparse/deformable storage — so the RL path plugs into existing sensing code, only the decision step is new. In gpu-resident mode, rl_policy.wgsl (a forward-pass-only WGSL port of the trained network's weights, refreshed after each train_step) lets inference run fully on-device for billion-agent swarms, matching the pattern V4's bt_jit.rs established for compiled trees.

## 7. src/orchestration/ — NEW: Heterogeneous Cloud Orchestration (Pillar #4)

V4's gpu::multi_gpu::MultiGpuSim shards a single voxel grid across a fixed local set of CUDA devices via NCCL. The roadmap's "serverless HPC task" framing goes further: the simulation itself shouldn't be bound to any particular cluster at all, and different subsystems have different ideal hardware — dense LBM collide/stream is Tensor-Core-friendly and belongs on H100s, while agent behavior-tree/GNN/RL logic and immune-response bookkeeping are branchy, low-arithmetic-intensity work that is often cheaper on ARM CPU pools or NPUs than on a GPU sitting idle waiting for scalar code.

// src/orchestration/worker.rs #![cfg(feature = "orchestration")] use crate::sim::params::HardwareTier; /// Uniform interface every hardware tier implements, so `scheduler.rs` /// never needs to know whether a segment is running on an H100 in one /// cloud region or an ARM CPU pool in another. #[async_trait::async_trait] pub trait Worker: Send + Sync { fn tier(&self) -> HardwareTier; } /// Opaque wire-format snapshots handed between the local orchestrator and /// remote workers over gRPC (`tonic`); actual layout mirrors the existing /// GPU-resident buffer layouts from `gpu/resident.rs` so no new /// serialization scheme is introduced. pub struct FluidStateBlob(pub Vec<u8>); A segment /// only migrates if the projected saving exceeds a hysteresis margin, so /// the scheduler doesn't thrash workloads between tiers over noise. pub struct HeterogeneousScheduler { workers: Vec<Box<dyn Worker>>, fluid_tier: HardwareTier, agent_tier: HardwareTier, migration_hysteresis: f64, // e.g. 0.15 = only migrate for >15% projected saving } impl HeterogeneousScheduler { pub fn new(workers: Vec<Box<dyn Worker>>, params: &crate::sim::params::Params) -> Self { HeterogeneousScheduler { workers, fluid_tier: params.fluid_hardware_tier, agent_tier: params.agent_hardware_tier, migration_hysteresis: 0.15, } } /// One orchestrated tick: dispatches the fluid-dynamics segment to /// whichever worker currently owns `fluid_tier` (defaults to CudaGpu — /// the roadmap's "high-end NVIDIA H100s for the fluid dynamics"), and /// the agent-logic/immune segment to whichever owns `agent_tier` /// (defaults to ArmCpu/Npu — "offloading ... to lower-cost ARM-based /// CPU clusters or specialized AI NPUs"). ## 8. src/testing/parity_suite.rs — NEW: Cross-Backend Parity Suite (V&V, Verification)

## 9. src/gpu/shader_verifier.rs — NEW: Formal Shader Verifier (V&V, Verification)

V4's bt_jit.rs generates WGSL by string templating (Improvement #7) — powerful, but with no guarantee the generated shader is race-free or memory-safe before it's handed to wgpu/the hardware driver. This module parses and statically analyzes every JIT-generated (and hand-written) shader with naga — the same WGSL front-end wgpu itself uses — catching storage-buffer race conditions and out-of-bounds indexing before dispatch, not after a corrupted simulation run.

#![cfg(feature = "shader-verify")] use naga::valid::{Validator, ValidationFlags, Capabilities}; use naga::front::wgsl; #[derive(Debug, Clone)] pub enum ShaderIssue { RaceCondition { binding: String, detail: String }, OutOfBoundsIndex { location: String, detail: String }, UnvalidatedControlFlow { detail: String }, } pub struct ShaderVerificationReport { pub source_name: String, pub issues: Vec<ShaderIssue>, } impl ShaderVerificationReport { pub fn is_safe(&self) -> bool { self.issues.is_empty() } } /// Parses `wgsl_source` into naga's IR and runs naga's own validator /// (bounds-checking, type-checking, control-flow validation) plus an /// additional MitoNano-specific race-condition scan over storage-buffer /// read_write bindings, since naga's validator alone does not reason about /// cross-invocation write ordering within a single dispatch. pub fn verify_shader(source_name: &str, wgsl_source: &str) -> ShaderVerificationReport { let mut issues = Vec::new(); ## 11. src/sim/stability.rs — NEW: Numerical Stability Monitoring / Divergence Detector (V&V, Verification)

At the billion-voxel scale V4's sparse/multi-GPU/precision-mixing subsystems target, small per-tick errors can compound into fluid-solver "explosions." This module monitors LBM entropy in real time and automatically rolls back, shrinks the timestep, and escalates precision for the specific sub-domain that's destabilizing — rather than letting a whole run silently corrupt or hard-crash.

#![cfg(feature = "stability")] use crate::sim::precision::PrecisionMode; /// Per-sub-domain stability state, tracked independently so a localized /// instability (e.g. right at a newly sprouted capillary from /// `morphogenesis.rs`) escalates precision only where it's needed, not /// across the whole voxel grid. pub struct StabilityMonitor { pub entropy_history: Vec<f64>, pub current_dt: f64, pub current_precision: PrecisionMode, pub rollback_count: u64, entropy_threshold: f64, dt_min: f64, dt_max: f64, } #[derive(Debug, Clone, Copy, PartialEq)] pub enum StabilityAction { Continue, RollbackAndShrinkDt { new_dt: f64 }, EscalatePrecision { new_mode: PrecisionMode }, RollbackShrinkAndEscalate { new_dt: f64, new_mode: PrecisionMode }, } impl StabilityMonitor { pub fn new(entropy_threshold: f64, dt_min: f64, dt_max: f64, initial_dt: f64) -> Self { StabilityMonitor { entropy_history: Vec::new(), current_dt: initial_dt, current_precision: PrecisionMode::Full, rollback_count: 0, entropy_threshold, dt_min, dt_max, } } /// Computes the Shannon entropy of the local distribution-function /// population (`f[slot*Q..slot*Q+Q]`, shared layout with V4's /// `sparse_lbm.rs`) — a standard LBM stability diagnostic, since a /// physically valid population stays close to the Maxwell-Boltzmann /// equilibrium's entropy and oscillates sharply just before numerical /// blow-up. pub fn compute_entropy(f_cell: &[f64]) -> f64 { let sum: f64 = f_cell.iter().sum(); f_cell.iter() .map(|&f| if f > 1e-12 { let p = f / sum; -p * p.ln() } else { 0.0 }) .sum() } /// Call once per tick per monitored sub-domain with this tick's mean /// entropy. Detects "oscillating unnaturally" (roadmap's phrasing) by /// checking the second difference of recent entropy samples against /// `entropy_threshold`, not just an absolute bound — genuine /// high-Reynolds-number turbulence has high entropy too, but it doesn't /// oscillate tick-to-tick the way an unstable numerical mode does. pub fn check(&mut self, mean_entropy: f64) -> StabilityAction { self.entropy_history.push(mean_entropy); if self.entropy_history.len() < 3 { return StabilityAction::Continue; } let n = self.entropy_history.len(); let second_diff = self.entropy_history[n - 1] - 2.0 * self.entropy_history[n - 2] + self.entropy_history[n - 3]; if second_diff.abs() > self.entropy_threshold { self.rollback_count += 1; let new_dt = (self.current_dt * 0.5).max(self.dt_min); let new_mode = escalate(self.current_precision); self.current_dt = new_dt; self.current_precision = new_mode; if new_mode != self.current_precision { StabilityAction::RollbackShrinkAndEscalate { new_dt, new_mode } } else { StabilityAction::RollbackAndShrinkDt { new_dt } } } else { // Slowly relax dt back toward dt_max when stable for a while, // so a transient instability doesn't permanently tax the whole // run's throughput once the cause (e.g. a remesh event) has // propagated through and settled. self.current_dt = (self.current_dt * 1.01).min(self.dt_max); StabilityAction::Continue } } } fn escalate(mode: PrecisionMode) -> PrecisionMode { match mode { PrecisionMode::MixedFp16 => PrecisionMode::MixedBf16, // wider dynamic range first PrecisionMode::MixedBf16 => PrecisionMode::Full, // then full f64/f32 PrecisionMode::Full => PrecisionMode::Full, // already at the ceiling } } 

Wiring into sim/engine.rs/sim/sparse_lbm.rs/sim/morphogenesis.rs: Simulation::step runs StabilityMonitor::check against every active FluidBackend's (dense, sparse, or deformable) per-tick entropy; a RollbackAndShrinkDt/RollbackShrinkAndEscalate result calls DeformableSparseGrid::rollback (§4, if morphogenesis was the proximate trigger) and re-runs the tick at the new dt/PrecisionMode, exactly matching the roadmap's "roll back the tick, reduce dt, increase precision ... until stability is restored." stats.rs (V4, extended) surfaces rollback_count and the entropy time series so a run that needed frequent rollbacks is visibly flagged in viz::plot's output, not silently absorbed.

## 12. src/sim/provenance.rs — NEW: Provenance & Traceability Tooling (Clinical Trust)

For a regulatory body to "re-run" a clinical claim (the roadmap's framing), every tuned-parameter file needs an unforgeable link back to the exact lab data and exact engine version that produced it — plain JSON (V1–V4's io::save_params) carries no such guarantee.

#![cfg(feature = "provenance")] use sha2::{Sha256, Digest}; use chrono::{DateTime, Utc}; use serde::{Serialize, Deserialize}; /// One append-only ledger entry: a cryptographic binding between a tuned /// parameter file, the lab data it was calibrated against, and the exact /// engine build that produced it. `entry_hash` chains to `prev_entry_hash` /// so the ledger itself is tamper-evident (a Merkle-chain, not just a flat /// list of independently-hashable records). #[derive(Serialize, Deserialize, Debug, Clone)] pub struct ProvenanceEntry { pub timestamp: DateTime<Utc>, pub engine_version: String, // e.g. "0.5.0" + git commit hash pub lab_data_hash: String, // sha256 of the IVIS dataset (sim/ivis.rs) used pub params_file_hash: String, // sha256 of the tuned params.json content pub calibration_backend: String, // "Evolutionary" | "GradientDescent" | "Adjoint" pub final_loss: f64, pub prev_entry_hash: String, pub entry_hash: String, } pub struct ProvenanceLedger { path: String, } impl ProvenanceLedger { pub fn new(path: &str) -> Self { ProvenanceLedger { path: path.to_string() } } /// Appends a new entry, computing `entry_hash` over every other field /// (including `prev_entry_hash`, read from the ledger's current last /// line) so any edit to an earlier entry breaks the chain from that /// point forward — the same tamper-evidence property a blockchain /// gives, without needing a distributed consensus layer for a /// single-institution audit trail. pub fn append( &mut self, engine_version: &str, lab_data: &[u8], params_file_content: &str, calibration_backend: &str, final_loss: f64, ) -> anyhow::Result<ProvenanceEntry> { let prev_entry_hash = self.last_entry_hash()?; let lab_data_hash = hex_sha256(lab_data); let params_file_hash = hex_sha256(params_file_content.as_bytes()); let timestamp = Utc::now(); let mut hasher = Sha256::new(); hasher.update(timestamp.to_rfc3339().as_bytes()); hasher.update(engine_version.as_bytes()); hasher.update(lab_data_hash.as_bytes()); hasher.update(params_file_hash.as_bytes()); hasher.update(calibration_backend.as_bytes()); hasher.update(final_loss.to_le_bytes()); hasher.update(prev_entry_hash.as_bytes()); let entry_hash = format!("{:x}", hasher.finalize()); let entry = ProvenanceEntry { timestamp, engine_version: engine_version.to_string(), lab_data_hash, params_file_hash, calibration_backend: calibration_backend.to_string(), final_loss, prev_entry_hash, entry_hash, }; self.write_line(&entry)?; Ok(entry) } /// Verifies the *entire* ledger's hash chain — used by a regulator's /// own tooling (or MitoNano's own CI) to confirm no entry has been /// edited or removed since it was written. pub fn verify_chain(&self) -> anyhow::Result<bool> { let entries = self.read_all()?; let mut prev = String::new(); for entry in &entries { if entry.prev_entry_hash != prev { return Ok(false); } let recomputed = recompute_entry_hash(entry); if recomputed != entry.entry_hash { return Ok(false); } prev = entry.entry_hash.clone(); } Ok(true) } fn last_entry_hash(&self) -> anyhow::Result<String> { Ok(self.read_all()?.last().map(|e| e.entry_hash.clone()).unwrap_or_default()) } fn read_all(&self) -> anyhow::Result<Vec<ProvenanceEntry>> { // Reads newline-delimited JSON (`.jsonl`) from `self.path`, one // `ProvenanceEntry` per line, tolerating a not-yet-created file. let _ = &self.path; Ok(Vec::new()) } fn write_line(&mut self, entry: &ProvenanceEntry) -> anyhow::Result<()> { let _ = entry; Ok(()) } } fn hex_sha256(bytes: &[u8]) -> String { let mut hasher = Sha256::new(); hasher.update(bytes); format!("{:x}", hasher.finalize()) } fn recompute_entry_hash(entry: &ProvenanceEntry) -> String { let mut hasher = Sha256::new(); hasher.update(entry.timestamp.to_rfc3339().as_bytes()); hasher.update(entry.engine_version.as_bytes()); hasher.update(entry.lab_data_hash.as_bytes()); hasher.update(entry.params_file_hash.as_bytes()); hasher.update(entry.calibration_backend.as_bytes()); hasher.update(entry.final_loss.to_le_bytes()); hasher.update(entry.prev_entry_hash.as_bytes()); format!("{:x}", hasher.finalize()) } 

Wiring into io::save_params/sim/calibration.rs: when Params::provenance_required is true, every successful AutoTuner/GradTuner/AdjointTuner/IvisCalibrationPipeline run must call ProvenanceLedger::append before io::save_params will write the tuned file at all — save_params is extended to refuse writing an un-ledgered file in this mode. A regulator (or MitoNano's own CI) can independently call ProvenanceLedger::verify_chain plus re-run IvisCalibrationPipeline against the hashed lab dataset to reproduce a claimed effectiveness figure exactly, matching the roadmap's "re-run that exact clinical trial in a sandbox environment to verify the results."

## 16. Suggested Next Steps (V6 candidates)

Extend DeformableSparseGrid from voxel-level solid/fluid flips to a full Lagrangian mesh deformation model (moving mesh vertices, not just re-tagging fixed-grid voxels), so vessel remodeling can capture geometric changes finer than one voxel's resolution.

Port AdjointSolver's tape recording fully onto the GPU (currently CPU-side faer matrices even when the forward pass itself runs GPU-resident), so multi-thousand-parameter inverse design at billion-voxel scale never round-trips the tape through host memory.

Extend RlAgentPolicy from single-agent PPO to a multi-agent/cooperative RL formulation (e.g. QMIX-style value decomposition), since swarm-level objectives (uniform drug distribution, coordinated hemorrhage plugging) are not always well-captured by summing independent single-agent rewards.

Extend HeterogeneousScheduler's cost model from a static per-tick estimate to a learned predictor trained on provenance.rs's own historical ledger data, closing the loop between "what did past runs actually cost" and "where should this run's segments be scheduled."

Extend ShaderVerificationReport's race-condition heuristic from a single-pass structural scan to a full symbolic execution of naga's IR across representative global_invocation_id ranges, catching races that only manifest for specific workgroup-size/dispatch-size combinations.

Wire sim::ivis::DigitalTwin beyond single-frame correlation into a full spatio-temporal trajectory comparison (optical flow between successive microscopy frames vs. simulated particle trajectories), since instantaneous concentration correlation can miss kinetic (rate) miscalibration even when steady-state distributions match.

Extend ProvenanceLedger from a local .jsonl hash chain to a distributed/notarized ledger (e.g. anchoring periodic root hashes to a public timestamping service), so provenance survives even a compromise of the institution's own storage, not just accidental edits.
