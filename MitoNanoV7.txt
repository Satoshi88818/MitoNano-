MitoNano Engine V7 — From Continuously-Self-Improving Clinical Platform to Multi-Physics, Multi-Notary Clinical Infrastructure

This document integrates the seven "Suggested Next Steps (V7 candidates)" from MitoNanoV6.txt §14 into the V6 architecture, producing a coherent V7 Rust workspace. As with every prior major version, all V1–V6 modules are preserved and extended, not replaced — existing params.json files keep deserializing (new fields carry #[serde(default)]), existing CLI invocations keep working, and every new subsystem is additive and feature-gated behind a new Cargo feature that layers on top of its V6 parent feature rather than replacing it.

Where V6's theme was fidelity, closure, and independence, V7's theme is heterogeneity, redundancy, and unification: tissue is no longer one uniform spring constant but many distinct materials, the adjoint tape no longer fits on one GPU, cooperation is no longer a fixed partition but a learned one, cost is no longer the only axis a scheduler must respect, race-freedom and memory-safety are no longer two separate proofs, kinetics are no longer flattened to a 2D projection, and trust in provenance no longer rests on any single notary.

4. src/sim/morphogenesis.rs — EXTENDED: Finite-Element Tissue Mechanics (V7 Pillar #1)

V6's LagrangianVesselMesh relaxes every vertex under the same mesh_spring_stiffness/mesh_damping pair, which is a reasonable approximation for a single homogeneous wall but cannot represent a tumor-invaded segment's friable, structurally distinct neovasculature sitting adjacent to healthy arterial tissue in the same mesh. TissueFemModel replaces the uniform spring network with a proper tetrahedral finite-element discretization: each element carries its own material properties (Young's modulus, Poisson's ratio) drawn from a MaterialRegion catalog, and the global stiffness matrix is assembled and solved per tick instead of relaxed with a fixed per-vertex spring law.

#![cfg(feature = "fem-tissue")] use crate::sim::morphogenesis::{LagrangianVesselMesh, VertexState}; use nalgebra::{Vector3, Matrix3}; use nalgebra_sparse::{CsrMatrix, CooMatrix}; use serde::{Serialize, Deserialize}; /// One tissue region's material properties, loaded from /// `Params::material_regions_path` — e.g. `{"healthy_wall": {young_modulus_kpa: /// 1200.0, poisson_ratio: 0.45}, "tumor_neovasculature": {young_modulus_kpa: /// 80.0, poisson_ratio: 0.49}}`. Each mesh element is tagged with a region /// id at import time (or at the last revoxelization), so a single mesh can /// mix healthy and pathological material behavior without the researcher /// hand-partitioning the geometry into separate meshes. #[derive(Clone, Debug, Serialize, Deserialize)] pub struct MaterialRegion { pub name: String, pub young_modulus_kpa: f64, pub poisson_ratio: f64, } /// One tetrahedral element's connectivity (into the mesh's vertex array) /// and its assigned material region — the FEM analogue of V6's per-vertex /// `rest_neighbors`, but element-based rather than edge-based, since strain /// energy is a property of a volume, not a spring. #[derive(Clone, Debug)] struct TetraElement { vertex_ids: [usize; 4], region: MaterialRegion, rest_volume: f64, } /// Continuous, heterogeneous-material tissue-mechanics model. Composes with /// V6's `LagrangianVesselMesh` exactly the way `LagrangianVesselMesh` /// composed with V5's `DeformableSparseGrid`: `mesh` remains the source of /// truth for geometry between revoxelizations, `elements` is the new /// tetrahedral discretization layered on top of the same vertex array, and /// `Params::tissue_mechanics_model == UniformSpring` bypasses this type /// entirely so V6 mass-spring behavior is byte-identical when not opted in. pub struct TissueFemModel { elements: Vec<TetraElement>, global_stiffness: CsrMatrix<f64>, // reassembled whenever material regions or topology change cg_tol: f64, cg_max_iters: u32, pub total_strain_energy: f64, // cumulative, surfaced via stats.rs per material region } impl TissueFemModel { pub fn from_mesh_and_regions( mesh: &LagrangianVesselMesh, region_tags: &[String], // one region name per tetrahedron, parallel to `elements` catalog: &[MaterialRegion], cg_tol: f64, cg_max_iters: u32, ) -> anyhow::Result<Self> { let elements = build_tetra_elements(mesh, region_tags, catalog)?; let global_stiffness = assemble_global_stiffness(&elements, mesh.vertex_positions().len()); Ok(TissueFemModel { elements, global_stiffness, cg_tol, cg_max_iters, total_strain_energy: 0.0 }) } /// Called every tick in place of V6's `relax()` when `TissueFemModel` /// is active: assembles the per-tick load vector from the same /// external-force sources V6's `relax()` consumed (angiogenic growth /// pressure, contact forces, wall shear), solves `K * du = f` via /// conjugate gradient for the incremental displacement `du`, and /// applies it to the mesh's vertex positions — this is the FEM /// analogue of one mass-spring semi-implicit Euler step, but the /// element-local stiffness contributions mean a stiff healthy-wall /// element and a soft tumor-neovasculature element genuinely deform /// at different rates under the same load, which a uniform spring /// constant could never represent. pub fn assemble_and_solve( &mut self, mesh: &mut LagrangianVesselMesh, external_forces: &[Vector3<f64>], dt: f64, ) -> anyhow::Result<()> { let load_vector = build_load_vector(external_forces, mesh.vertex_positions().len()); Ok(()) } /// Called from `LagrangianVesselMesh::maybe_revoxelize`'s existing /// cadence (`Params::mesh_revoxelize_every`) when tissue topology /// itself changes (angiogenic sprouting adds vertices/elements) — /// re-derives `elements`' connectivity and reassembles /// `global_stiffness` from scratch, since a sparse stiffness matrix's /// structure depends on element connectivity and cannot be /// incrementally patched across a topology change. pub fn reassemble_after_revoxelize(&mut self, mesh: &LagrangianVesselMesh, region_tags: &[String], catalog: &[MaterialRegion]) -> anyhow::Result<()> { self.elements = build_tetra_elements(mesh, region_tags, catalog)?; self.global_stiffness = assemble_global_stiffness(&self.elements, mesh.vertex_positions().len()); Ok(()) } } fn build_tetra_elements(mesh: &LagrangianVesselMesh, region_tags: &[String], catalog: &[MaterialRegion]) -> anyhow::Result<Vec<TetraElement>> { // Tetrahedralizes the mesh's current triangle surface (parry3d-f64, // already present) into a volumetric mesh, tags each tetrahedron with // its `region_tags` entry resolved against `catalog`, and records // each element's rest volume for later strain-energy accounting. let _ = (mesh, region_tags, catalog); unimplemented!("Delaunay tetrahedralization of the surface mesh + region-tag resolution against `catalog`") } fn assemble_global_stiffness(elements: &[TetraElement], n_vertices: usize) -> CsrMatrix<f64> { // Standard FEM assembly loop: for each element, computes its 12x12 // local stiffness matrix from the element's material properties // (via the isotropic linear-elasticity constitutive law parameterized // by young_modulus_kpa/poisson_ratio) and scatter-adds it into a COO // accumulator at the global degree-of-freedom indices, converting to // CSR once at the end for fast matrix-vector products during CG. let mut coo = CooMatrix::new(3 * n_vertices, 3 * n_vertices); for element in elements { let local_k = element_stiffness_matrix(element); scatter_add_local_stiffness(&mut coo, element, &local_k); } CsrMatrix::from(&coo) } fn element_stiffness_matrix(element: &TetraElement) -> nalgebra::SMatrix<f64, 12, 12> { // Isotropic linear elasticity: builds the element's strain-displacement // (B) and constitutive (D, from young_modulus_kpa/poisson_ratio) matrices // and forms K_e = V_e * B^T * D * B, the standard constant-strain- // tetrahedron stiffness formula. let _ = element; nalgebra::SMatrix::<f64, 12, 12>::zeros() } fn scatter_add_local_stiffness(coo: &mut CooMatrix<f64>, element: &TetraElement, local_k: &nalgebra::SMatrix<f64, 12, 12>) { let _ = (coo, element, local_k); // Adds each of the 12x12 local entries into the global COO matrix at // the (3*vertex_id + component) indices for the element's 4 vertices. } fn build_load_vector(external_forces: &[Vector3<f64>], n_vertices: usize) -> Vec<f64> { let mut f = vec![0.0; 3 * n_vertices]; for (i, force) in external_forces.iter().enumerate().take(n_vertices) { f[3 * i] = force.x; f[3 * i + 1] = force.y; f[3 * i + 2] = force.z; } f } fn conjugate_gradient_solve(k: &CsrMatrix<f64>, f: &[f64], tol: f64, max_iters: u32) -> anyhow::Result<Vec<f64>> { let _ = (k, f, tol, max_iters); unimplemented!("standard CG on K*du = f; GPU path below runs the same iteration as fem_solve.wgsl") } fn apply_displacement_to_mesh(mesh: &mut LagrangianVesselMesh, du: &[f64], dt: f64) { let _ = (mesh, du, dt); // Applies the solved incremental displacement to each vertex's // position (and derives an implied velocity for consistency with // V6's VertexState.velocity field, so BVH refit's velocity-threshold // trigger continues to work unchanged). } fn strain_energy(elements: &[TetraElement], du: &[f64]) -> f64 { let _ = (elements, du); 0.0 // sum over elements of 0.5 * du_e^T * K_e * du_e } 

GPU path (gpu/shaders/fem_assemble.wgsl and gpu/shaders/fem_solve.wgsl, both new): element-local stiffness assembly is embarrassingly parallel across tetrahedra (one workgroup thread per element, following the same one-thread-per-element pattern as V6's lagrangian_relax.wgsl), so fem_assemble.wgsl computes every element's local 12×12 stiffness contribution on-device; the scatter-add into the global sparse matrix and the subsequent CG solve run as fem_solve.wgsl, a standard matrix-free CG iteration (Ax implemented as an element-local gather-scatter rather than materializing the full sparse matrix on-device, avoiding a CSR-on-GPU representation entirely). Both shaders are routed through gpu/shader_verifier.rs before dispatch, per V5/V6's shader-verification gate — and, when Params::shader_proof_scope == RaceAndBounds (V7 Pillar #5), through the unified verifier, since the scatter-add step is exactly the kind of indexed write where an out-of-bounds element index would previously have gone unproven.

Wiring into sim/morphogenesis.rs/sim/engine.rs: LagrangianVesselMesh gains an optional fem: Option<TissueFemModel> field, constructed from Params::material_regions_path when Params::tissue_mechanics_model == TissueMechanicsModel::Fem. 6. src/sim/rl_agent.rs — EXTENDED: Attention-Based Dynamic QMIX Grouping (V7 Pillar #3)

8. src/gpu/shader_verifier.rs — EXTENDED: Unified Race + Bounds Symbolic Verifier (V7 Pillar #5)

V6's SymbolicExecutor proves race-freedom as one proof obligation and leaves out-of-bounds storage-buffer access as a separate, unaddressed question — a shader could be formally proven race-free and still read past the end of a buffer for some symbolic global_invocation_id value the structural pass never considered. 9. src/sim/ivis.rs — EXTENDED: 3D Particle-Tracking Velocimetry (V7 Pillar #6)

V6's TrajectoryComparator computes optical flow over a 2D projection of the imaging plane — adequate for widefield or single-plane confocal data, but a 2D projection can alias two genuinely different 3D trajectories (a particle moving diagonally through the imaging volume and one moving laterally within a single plane can produce near-identical 2D projected flow). VolumetricTrajectoryComparator performs true 3D particle-tracking velocimetry (PTV) when the digital-twin dataset includes light-sheet or confocal volumetric time series: individual particles are detected per volume frame and linked across consecutive frames via nearest-neighbor search in the full 3D position space, rather than the dense-flow-field approach V6 used for 2D.

#![cfg(feature = "ivis-3d-ptv")] use kiddo::{KdTree, distance::squared_euclidean}; use ndarray::Array3; use crate::sim::ivis::{DigitalTwin, MicroscopyVolumeFrame}; /// One detected particle in a single volumetric frame — the 3D-PTV /// analogue of a 2D optical-flow field's per-pixel vector, except PTV is /// inherently sparse (individual particle positions, not a dense per-voxel /// field), which is what makes nearest-neighbor linking the right /// correspondence method rather than a dense solve. #[derive(Clone, Debug)] pub struct DetectedParticle3D { pub position: [f64; 3], // imaging-volume physical units pub intensity: f64, pub frame_index: usize, } /// A linked trajectory segment between two consecutive volume frames — /// one particle's inferred 3D displacement, the ground truth PTV compares /// against the simulated side's `SimulatedTrajectorySegment` (V6, reused /// unchanged — only its *projection* step is bypassed in 3D mode, since /// the simulated displacement no longer needs to be projected onto a 2D /// imaging plane when the comparison itself is fully volumetric). #[derive(Clone, Debug)] pub struct LinkedTrajectory3D { pub start: DetectedParticle3D, pub end: DetectedParticle3D, pub displacement: [f64; 3], } /// Detects particle centroids in one volumetric frame — a local-maxima /// search over the intensity field above a noise floor, refined to /// sub-voxel precision via a small Gaussian-fit around each maximum /// (the standard PTV detection approach for diffraction-limited spots in /// light-sheet/confocal data). pub fn detect_particles_3d(volume: &Array3<f64>, frame_index: usize, noise_floor: f64) -> Vec<DetectedParticle3D> { let _ = (volume, frame_index, noise_floor); unimplemented!("local-maxima detection + sub-voxel Gaussian centroid refinement over the 3D intensity field") } /// Links particles between two consecutive frames via nearest-neighbor /// search in 3D position space, using a k-d tree (`kiddo`) built over /// `frame_b`'s detections — restricting candidate matches to /// `Params::ptv_link_radius` of a particle's frame-A position (rather than /// a global nearest-neighbor search) keeps linking well-posed even when /// particle density is high enough that the globally-nearest particle in /// frame B is not actually the same physical particle. pub fn link_particles_3d( frame_a_particles: &[DetectedParticle3D], frame_b_particles: &[DetectedParticle3D], link_radius: f64, ) -> Vec<LinkedTrajectory3D> { let mut tree: KdTree<f64, usize, 3> = KdTree::new(); for (i, p) in frame_b_particles.iter().enumerate() { let _ = tree.add(&p.position, i); } let mut linked = Vec::new(); for particle_a in frame_a_particles { if let Ok(nearest) = tree.nearest_one(&particle_a.position, &squared_euclidean) { let (dist_sq, &idx) = nearest; if dist_sq.sqrt() <= link_radius { let particle_b = &frame_b_particles[idx]; let displacement = [ particle_b.position[0] - particle_a.position[0], particle_b.position[1] - particle_a.position[1], particle_b.position[2] - particle_a.position[2], ]; linked.push(LinkedTrajectory3D { start: particle_a.clone(), end: particle_b.clone(), displacement }); } } } linked } /// Extends V6's `TrajectoryComparator` to full volumetric comparison. Kept /// as a separate type composed alongside `DigitalTwin` (not a modification /// to `TrajectoryComparator`), per the same additive rule V6 followed for /// its own 2D comparator, so `Params::ivis_trajectory_dimensionality == /// Projected2D` callers are completely unaffected. pub struct VolumetricTrajectoryComparator<'a> { twin: &'a DigitalTwin, link_radius: f64, drift_threshold: f64, } #[derive(Debug)] pub struct VolumetricComparisonReport { pub per_frame_pair_drift: Vec<f64>, pub flagged_kinetic_kernels: Vec<String>, pub mean_linking_accuracy: f64, // fraction of detections successfully linked, surfaced via stats.rs } impl<'a> VolumetricTrajectoryComparator<'a> { pub fn new(twin: &'a DigitalTwin, link_radius: f64, drift_threshold: f64) -> Self { VolumetricTrajectoryComparator { twin, link_radius, drift_threshold } } /// Walks consecutive volumetric frame pairs, detects and links /// particles in true 3D, and compares each linked trajectory's /// displacement against the matching simulated particle's actual 3D /// displacement (no 2D projection on either side) — a per-pair drift /// score above `Params::trajectory_drift_threshold` implicates the /// kinetic kernel exactly as V6's 2D comparator did, but is no longer /// vulnerable to two distinct 3D trajectories aliasing under a shared /// 2D projection. pub fn compare( &self, volume_frames: &[MicroscopyVolumeFrame], sim_segments_3d: &[(usize, Vec<[f64; 3]>)], // (frame_pair_index, per-particle 3D displacement) ) -> VolumetricComparisonReport { let mut per_frame_pair_drift = Vec::new(); let mut flagged_kinetic_kernels = Vec::new(); let mut total_detections = 0usize; let mut total_linked = 0usize; for pair_index in 0..volume_frames.len().saturating_sub(1) { let particles_a = detect_particles_3d(&volume_frames[pair_index].intensity, pair_index, self.twin.noise_floor()); let particles_b = detect_particles_3d(&volume_frames[pair_index + 1].intensity, pair_index + 1, self.twin.noise_floor()); total_detections += particles_a.len(); let linked = link_particles_3d(&particles_a, &particles_b, self.link_radius); total_linked += linked.len(); if let Some((_, sim_displacements)) = sim_segments_3d.iter().find(|(i, _)| *i == pair_index) { let drift = trajectory_disagreement_3d(&linked, sim_displacements); per_frame_pair_drift.push(drift); if drift > self.drift_threshold { flagged_kinetic_kernels.push(implicated_kinetic_kernel_3d(pair_index)); } } } let mean_linking_accuracy = if total_detections > 0 { total_linked as f64 / total_detections as f64 } else { 0.0 }; VolumetricComparisonReport { per_frame_pair_drift, flagged_kinetic_kernels, mean_linking_accuracy } } } fn trajectory_disagreement_3d(linked: &[LinkedTrajectory3D], sim_displacements: &[[f64; 3]]) -> f64 { let _ = (linked, sim_displacements); 0.0 // normalized combination of 3D angular disagreement + magnitude ratio, averaged over linked particles } fn implicated_kinetic_kernel_3d(frame_pair_index: usize) -> String { let _ = frame_pair_index; "diffusion_coefficient".to_string() // or "flow_coupling_term" / "z_axis_drift_term" (new in 3D — a term 2D projection couldn't even express) } 

Wiring into sim/ivis.rs/sim/calibration.rs: IvisCalibrationPipeline::run (V5/V6) checks Params::ivis_trajectory_dimensionality alongside V6's existing ivis_comparison_mode check — Projected2D runs V6's TrajectoryComparator exactly as before, Volumetric3D (which implies ivis_comparison_mode == Trajectory, since 3D PTV is a trajectory-level technique) constructs VolumetricTrajectoryComparator instead and unions its flagged_kinetic_kernels into the same combined set V6 already hands to AdjointTuner/GradTuner, plus a new possible kernel — z_axis_drift_term — that a 2D projection could never have named in the first place, since it has no representation in a 2D imaging plane at all.

10. src/sim/provenance.rs — EXTENDED: Redundant Multi-Notary Anchoring (V7 Pillar #7)

V6's MerkleAnchor notarizes each batch's root hash with exactly one external Time-Stamping Authority — a real improvement over pure local hash-chaining, but a single point of failure: if that one TSA is compromised, shuts down, or is later found to have colluded with the institution, every anchor it issued becomes unverifiable. MultiNotaryAnchor fans the same Merkle root out to Params::notary_endpoints, and an anchor is only considered externally attested once at least Params::notary_quorum of them independently corroborate it — provenance now survives not just a compromise of the institution's own storage (V6's guarantee) but the failure or compromise of any single external notary as well.

#![cfg(feature = "provenance-multi-notary")] use rs_merkle::{MerkleTree, algorithms::Sha256 as MerkleSha256, Hasher}; use serde::{Serialize, Deserialize}; use crate::sim::provenance::{ProvenanceEntry, MerkleAnchor, AnchorRecord}; /// One notary's individual attestation for a batch — structurally /// identical to V6's single `AnchorRecord`, just tagged with which /// endpoint produced it, so a quorum can be counted per-batch across /// however many of `Params::notary_endpoints` actually responded in time. #[derive(Serialize, Deserialize, Debug, Clone)] pub struct NotaryAttestation { pub notary_endpoint: String, pub record: AnchorRecord, pub responded: bool, // false if this notary timed out / errored for this batch } /// A batch's *combined* multi-notary result — the V7 unit of provenance /// trust, replacing V6's single `AnchorRecord` as what /// `ProvenanceLedger::verify_chain` actually checks against when /// `Params::provenance_anchor_policy == RedundantQuorum`. #[derive(Serialize, Deserialize, Debug, Clone)] pub struct QuorumAnchorRecord { pub batch_start_entry_hash: String, pub batch_end_entry_hash: String, pub merkle_root: String, pub attestations: Vec<NotaryAttestation>, pub quorum_met: bool, // attestations.iter().filter(|a| a.responded).count() >= Params::notary_quorum } /// Fans one batch's anchoring request out to every configured notary /// endpoint. Composed alongside V6's `MerkleAnchor` (reused internally, /// one instance per endpoint) rather than replacing it — a /// `notary_endpoints` list of length 1 with `notary_quorum == 1` /// reproduces V6's single-notary behavior exactly, so `RedundantQuorum`'s /// degenerate case is byte-for-byte `SingleNotary`. pub struct MultiNotaryAnchor { notaries: Vec<MerkleAnchor>, // one per Params::notary_endpoints entry quorum: usize, } impl MultiNotaryAnchor { pub fn new(notary_endpoints: &[String], batch_size: usize, quorum: usize) -> Self { let notaries = notary_endpoints.iter().map(|ep| MerkleAnchor::new(ep, batch_size)).collect(); MultiNotaryAnchor { notaries, quorum } } /// Requests an attestation from every configured notary independently /// — a slow or failed notary does not block the batch (its /// `NotaryAttestation::responded` is simply `false`), since the whole /// point of redundancy is that no single notary's availability gates /// the system; `quorum_met` is computed from however many *did* /// respond in time, which the batch's own retry logic (below) can /// later top up if the initial quorum wasn't met. pub fn anchor_batch_redundant(&self, batch_entries: &[ProvenanceEntry]) -> anyhow::Result<QuorumAnchorRecord> { let leaves: Vec<[u8; 32]> = batch_entries.iter().map(|e| MerkleSha256::hash(e.entry_hash.as_bytes())).collect(); let tree = MerkleTree::<MerkleSha256>::from_leaves(&leaves); let root = tree.root().ok_or_else(|| anyhow::anyhow!("empty batch"))?; let merkle_root = hex::encode(root); let mut attestations = Vec::with_capacity(self.notaries.len()); for notary in &self.notaries { match notary.anchor_batch(batch_entries) { Ok(record) => attestations.push(NotaryAttestation { notary_endpoint: notary.endpoint().to_string(), record, responded: true }), Err(_) => attestations.push(NotaryAttestation { notary_endpoint: notary.endpoint().to_string(), record: AnchorRecord { batch_start_entry_hash: batch_entries.first().unwrap().entry_hash.clone(), batch_end_entry_hash: batch_entries.last().unwrap().entry_hash.clone(), merkle_root: merkle_root.clone(), tsa_token: Vec::new(), anchored_at: chrono::Utc::now(), }, responded: false, }), } } let responded_count = attestations.iter().filter(|a| a.responded).count(); Ok(QuorumAnchorRecord { batch_start_entry_hash: batch_entries.first().unwrap().entry_hash.clone(), batch_end_entry_hash: batch_entries.last().unwrap().entry_hash.clone(), merkle_root, attestations, quorum_met: responded_count >= self.quorum, }) } /// Independent verification path: re-checks every attestation that /// *did* respond against its own notary's public verification /// endpoint (`MerkleAnchor::verify_anchor`, V6, unchanged, called once /// per attestation), and additionally confirms the *quorum count* /// itself — a regulator running this needs to trust neither /// MitoNano's server nor any single notary, only that at least /// `quorum` independently-operated notaries corroborate the same root. pub fn verify_quorum(&self, record: &QuorumAnchorRecord) -> anyhow::Result<bool> { let mut verified_count = 0usize; for (attestation, notary) in record.attestations.iter().zip(self.notaries.iter()) { if attestation.responded && notary.verify_anchor(&attestation.record)? { verified_count += 1; } } Ok(verified_count >= self.quorum) } /// Retries anchoring against only the notaries that failed to respond /// in `record` — called by `ProvenanceLedger::append`'s background /// retry path (extended below) when `quorum_met` was initially false, /// so a transient outage at one or two notaries doesn't permanently /// leave a batch under-attested if those notaries recover. pub fn retry_missing(&self, record: &mut QuorumAnchorRecord, batch_entries: &[ProvenanceEntry]) -> anyhow::Result<()> { for (attestation, notary) in record.attestations.iter_mut().zip(self.notaries.iter()) { if !attestation.responded { if let Ok(new_record) = notary.anchor_batch(batch_entries) { attestation.record = new_record; attestation.responded = true; } } } let responded_count = record.attestations.iter().filter(|a| a.responded).count(); record.quorum_met = responded_count >= self.quorum; Ok(()) } } 

Wiring into sim/provenance.rs's ProvenanceLedger: append (V5, extended in V6) is extended once more so that, when Params::provenance_anchor_policy == ProvenanceAnchorPolicy::RedundantQuorum, every anchor_batch_size-th append calls MultiNotaryAnchor::anchor_batch_redundant instead of V6's single-notary anchor_batch, appending the resulting QuorumAnchorRecord to the same <ledger_path>.anchors.jsonl file (a superset-shaped record, so V6's anchor-reading tooling that only inspects merkle_root/anchored_at keeps working unmodified). ProvenanceLedger::verify_chain is extended to call verify_quorum for every recorded batch boundary under RedundantQuorum, so a full audit confirms the local hash chain is internally consistent (V5), that it is externally notarized (V6), and that the notarization itself does not depend on any single notary's continued honesty or availability (V7) — a batch anchored under a 2-of-3 quorum remains provably attested even if one of the three notaries is later discovered to have been compromised for the entire anchoring period, since the other two's independent corroboration stands on its own.

14. Suggested Next Steps (V8 candidates)

Extend TissueFemModel from linear (small-strain) elasticity to a hyperelastic (finite-strain) constitutive model for vessel walls under large deformation, since angiogenic sprouting and surgical retraction routinely exceed the strain regime where a linear stress-strain relationship stays accurate.

Extend ShardedGpuTapeBuffer from a static shard-per-device assignment fixed at construction to dynamic re-sharding that responds to orchestration::multi_gpu's own runtime heterogeneity signals (a device falling behind mid-run should shed shards, not just have been sized smaller up front).

Extend AttentionGroupingNet from scoring pairwise coupling over the current tick's observations alone to incorporating short-horizon coupling history, so a cluster assignment reflects sustained coupling rather than a single tick's possibly-noisy snapshot.

Extend LatencyAwareCostModel from two independent regression heads to a model that also captures the cost-latency trade-off curve itself (predicting a Pareto frontier, not two point estimates), so the scheduler can pick an explicit operating point rather than a hard latency cutoff.

Extend UnifiedSymbolicExecutor from per-binding joint proofs to whole-pipeline symbolic execution across chained compute-shader dispatches, proving race-freedom and bounds-safety hold not just within one shader but across a multi-stage GPU pipeline's inter-stage buffer handoffs.

Extend VolumetricTrajectoryComparator from nearest-neighbor frame-to-frame linking to a proper multi-frame PTV tracking algorithm (e.g. a Kalman-filtered trajectory model) that survives brief particle occlusion or missed detections, since two-frame nearest-neighbor linking silently breaks a trajectory the moment one frame's detection is missed.

Extend MultiNotaryAnchor from a fixed, operator-configured notary list to a governance model where the notary set itself is subject to auditable rotation and addition/removal, so provenance trust doesn't quietly degrade to a stale quorum if one of the original notaries goes permanently offline years after a batch was anchored.
