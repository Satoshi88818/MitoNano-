MitoNano Engine V11 — From Self-Remodeling, Correlation-Aware, Anticipatory, State-Conditioned, Liveness-Proven, Kinetically-Grounded, Dual-Authority Clinical Infrastructure to Neighbor-Coupled, Factor-Correlated, Regime-Targeted, Trajectory-Conditioned, Branch-Fair, Catalog-Adaptive, Federated-Authority Clinical Infrastructure

This document integrates the seven "Suggested Next Steps (V11 candidates)" from MitoNanoV10.txt §13 into the V10 architecture, producing a coherent V11 Rust workspace. As with every prior major version, all V1–V10 modules are preserved and extended, not replaced — existing params.json files keep deserializing (new fields carry #[serde(default)]), existing CLI invocations keep working, and every new subsystem is additive and feature-gated behind a new Cargo feature that layers on top of its V10 parent feature rather than replacing it.

Where V10's theme was closing the gap between "the model reacts correctly" and "the model is honest about its own drift," V11's theme is closing the gap between "the model is locally correct" and "the model is correct in context": a fiber ODE is no longer blind to its neighbors, a correlation estimate is no longer inferred indirectly from pairwise coefficients, a regime-change head no longer predicts that a shift is coming without predicting what it's a shift to, a frontier conditioning vector no longer collapses a trajectory into a single instant, a liveness bound no longer applies one number to branches with structurally different fairness expectations, an IMM catalog is no longer frozen at run start, and an external-authority gate is no longer a single point of failure.

3. Pillar #1 — Neighbor-Coupled Fiber Remodeling (sim/morphogenesis.rs)

Problem it closes: RemodelingAnisotropicHyperelasticModel (V10) integrates a per-element reorientation ODE driven only by that element's locally sensed stress tensor. Adjacent elements under similar loads can converge to noticeably different fiber angles — a numerically stable but physically implausible sawtooth field, since real collagen bundles mechanically couple to their neighbors through shared load paths.

Fix: add a diffusion term over a NeighborFiberCouplingGraph (mesh adjacency), turning the per-element ODE into a coupled reaction-diffusion PDE. DiffusionCoupledRemodelingModel composes V10's model as base and adds the coupling term on top; setting the coupling coefficient to 0.0 recovers V10 exactly.

// sim/morphogenesis.rs — EXTENDED use petgraph::graph::{UnGraph, NodeIndex}; /// V10's per-element model, unchanged. pub use crate::sim::morphogenesis::RemodelingAnisotropicHyperelasticModel; /// Mesh-adjacency graph used purely for fiber-coupling; pub type ElementId = u32; impl NeighborFiberCouplingGraph { /// Build from FEM face-adjacency, weighting edges by shared-face area /// so a coarse element that borders a fine element on a small sliver /// couples more weakly than one sharing a full face. pub fn from_face_adjacency(mesh: &FemMesh) -> Self { let mut graph = UnGraph::new_undirected(); let mut element_index = std::collections::HashMap::new(); for elem in mesh.elements() { let idx = graph.add_node(elem.id); element_index.insert(elem.id, idx); } for (a, b, shared_face_area) in mesh.face_adjacency_pairs() { let (ia, ib) = (element_index[&a], element_index[&b]); 0.0 recovers V10 exactly } impl DiffusionCoupledRemodelingModel { pub fn step( &mut self, state: &mut FiberRemodelingState, stress_field: &ElementStressField, dt_ticks: f64, ) -> DiffusionStepReport { // 1. V10's local reorientation delta, one call per element, unchanged. let base_deltas = self.base.compute_local_deltas(state, stress_field, dt_ticks); // 2. V11's neighbor-coupling correction, added on top. let mut coupling_deltas = vec![0.0_f64; state.theta.len()]; let mut max_neighbor_disagreement = 0.0_f64; for (&elem_id, &node) in &self.coupling_graph.element_index { let theta_i = state.theta[&elem_id]; let mut weighted_sum = 0.0; let mut weight_total = 0.0; for edge in self.coupling_graph.graph.edges(node) { let neighbor_id = self.coupling_graph.graph[edge.target()]; let theta_j = state.theta[&neighbor_id]; let w = *edge.weight(); let diff = wrap_angle_diff(theta_j - theta_i); weighted_sum += w * diff; weight_total += w; max_neighbor_disagreement = max_neighbor_disagreement.max(diff.abs()); } let normalized = if weight_total > 0.0 { weighted_sum / weight_total } else { 0.0 }; coupling_deltas[node.index()] = self.coupling_coefficient * normalized * dt_ticks; } // 3. Combine and integrate. for (elem_id, node) in &self.coupling_graph.element_index { let base_d = base_deltas[elem_id]; let coup_d = coupling_deltas[node.index()]; *state.theta.get_mut(elem_id).unwrap() += base_d + coup_d; } DiffusionStepReport { max_neighbor_disagreement_rad: max_neighbor_disagreement, mean_coupling_delta_rad: coupling_deltas.iter().sum::<f64>() / coupling_deltas.len().max(1) as f64, } } } fn wrap_angle_diff(d: f64) -> f64 { use std::f64::consts::PI; let mut x = d % PI; if x > PI / 2.0 { x -= PI; } if x <= -PI / 2.0 { x += PI; } x } pub struct DiffusionStepReport { pub max_neighbor_disagreement_rad: f64, pub mean_coupling_delta_rad: f64, }

gpu/shaders/fem_assemble_anisotropic_remodeling_diffusion.wgsl mirrors this: the V10 shader's per-element reorientation kernel is unchanged and called first into a scratch buffer; a second compute pass reads each element's precomputed neighbor list (built once at mesh load, uploaded as a CSR-format buffer) and adds the graph-Laplacian correction, avoiding a second full FEM assembly pass.

4. Pillar #2 — Low-Rank Shared-Factor Correlation (gpu/adjoint_gpu.rs, sim/adjoint.rs)

Problem it closes: V10's CrossDeviceCorrelationEstimator computes pairwise Pearson correlation between every pair of devices' forecast residuals. This catches two-device co-degradation but (a) scales as O(n²) pairs, and (b) a genuinely cluster-wide shock — e.g. a facility cooling event nudging every device's thermal throttling residual up together — shows up as every pair being moderately correlated rather than any single pair standing out, so it has to be inferred indirectly from a dense pairwise matrix instead of being visible directly.

Fix: replace the estimation target with a low-rank shared-latent-factor model, fit by streaming incremental PCA over the residual matrix. A cluster-wide shock becomes a single dominant factor loading across all devices — directly interpretable — rather than a fully dense pairwise matrix.

// gpu/adjoint_gpu.rs — EXTENDED /// V10, unchanged; still available directly and used as the estimator's /// diagnostic fallback (its output is retained purely for comparison/back- /// compat plotting, not consumed by the migration planner under V11 mode). pub use crate::gpu::adjoint_gpu::CrossDeviceCorrelationEstimator; /// V11: incremental low-rank factor model over per-device forecast residuals. /// /// Model: r_t (n_devices-vector of residuals at tick t) ≈ L * f_t + ε_t /// where L is an (n_devices × k) loading matrix and f_t is a k-vector of /// shared latent factors. 5. Pillar #3 — Regime-Targeted Window Anticipation (sim/rl_agent.rs)

Problem it closes: V10's AnticipatoryTemporalAttentionGroupingNet adds a scalar shift-probability head to V9's reactive controller, letting it trigger reclustering before a regime change instead of after. But the new window length is still computed by extrapolating the pre-shift encoding — the head knows a shift is coming, not what it's a shift to.

Fix: add a second head predicting the target regime's characteristic volatility level, and use it (once the shift-probability crosses the trigger threshold) to select the pre-emptive window length directly, instead of extrapolating the ending regime.

6. Pillar #4 — Trajectory-Conditioned Pareto Frontier (orchestration/cost_model.rs, orchestration/multi_gpu.rs)

Problem it closes: StateConditionedParetoFrontierModel (V10) conditions the frontier fit on the instantaneous system-state vector (queue depth, thermal headroom, time of day). A queue depth of 40 that has climbed steadily for ten ticks and a queue depth of 40 that has fallen for ten ticks are the same point-in-time feature but predict different frontiers a few ticks out.

Fix: extend the conditioning feature vector from a point to a short trajectory summary (current value + slope + curvature over a rolling window), computed cheaply from multi_gpu.rs's now-retained rolling state history.

7. Pillar #5 — Per-Branch Fair Liveness Proof (gpu/shader_verifier.rs)

Problem it closes: LivenessProvingPipelineSymbolicExecutor (V10) proves the scheduler's dispatch-graph policy must eventually reach some proved-safe branch, under a single global fairness bound. Branches are not equally likely by design — a fallback-only path should legitimately be selected less often than a hot path — so one global bound either flags legitimate rare-but-fair branches as violations, or is loosened enough to miss genuine starvation on a branch that should be common.

Fix: attach a fairness bound to each branch in the dispatch graph rather than one bound to the whole graph, sourced from a declared per-branch "expected selection class" (hot / warm / fallback) instead of a single number.

// gpu/shader_verifier.rs — EXTENDED /// V10, unchanged; still runs first as the safety pass, and V11's liveness /// executor composes it as `base` exactly as V10 composed the V9 executor. pub use crate::gpu::shader_verifier::{DynamicPipelineSymbolicExecutor, LivenessObligation}; #[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize)] pub enum BranchFairnessClass { Hot, // expected to be selected on the large majority of dispatches Warm, // expected occasionally, under specific but non-rare conditions Fallback, // expected rarely, only under exceptional/degraded conditions } /// Params::shader_proof_obligation_scope::PerBranchFair carries a per-class /// fairness bound instead of V10's single ShaderLivenessFairnessBoundTicks. #[derive(Debug, Clone, serde::Serialize, serde::Deserialize)] pub struct PerBranchFairnessBounds { pub hot_bound_ticks: u64, pub warm_bound_ticks: u64, pub fallback_bound_ticks: u64, } impl PerBranchFairnessBounds { pub fn bound_for(&self, class: BranchFairnessClass) -> u64 { match class { BranchFairnessClass::Hot => self.hot_bound_ticks, BranchFairnessClass::Warm => self.warm_bound_ticks, BranchFairnessClass::Fallback => self.fallback_bound_ticks, } } } /// Each dispatch-graph branch is now annotated with a fairness class at /// graph-construction time (declared by the scheduler policy author, not /// inferred), so this table is fixed input to verification, not fit data. pub struct BranchFairnessAnnotations(pub std::collections::HashMap<BranchId, BranchFairnessClass>); /// V11: composes V10's executor as `base`. The safety pass and the "some /// branch is eventually reachable" liveness pass are delegated unchanged; /// V11 adds a second liveness-obligation discharge pass that checks each /// branch's *own* fairness bound rather than one shared bound. pub struct PerBranchFairLivenessSymbolicExecutor { pub base: crate::gpu::shader_verifier::LivenessProvingPipelineSymbolicExecutor, // V10 pub bounds: PerBranchFairnessBounds, pub annotations: BranchFairnessAnnotations, } pub struct PerBranchLivenessReport { pub base_report: crate::gpu::shader_verifier::LivenessReport, // V10, unchanged shape pub per_branch_discharge: std::collections::HashMap<BranchId, BranchDischargeResult>, } pub enum BranchDischargeResult { Discharged { max_wait_ticks_observed: u64, bound_ticks: u64 }, Violated { max_wait_ticks_observed: u64, bound_ticks: u64, class: BranchFairnessClass }, } impl PerBranchFairLivenessSymbolicExecutor { pub fn verify(&self, dispatch_graph: &DispatchGraph) -> PerBranchLivenessReport { // V10's safety + "eventually some branch" liveness pass, unchanged. let base_report = self.base.verify(dispatch_graph); let mut per_branch_discharge = std::collections::HashMap::new(); for (&branch_id, &class) in &self.annotations.0 { let bound = self.bounds.bound_for(class); let observed_max_wait = self.base.max_wait_ticks_for_branch(dispatch_graph, branch_id); let result = if observed_max_wait <= bound { BranchDischargeResult::Discharged { max_wait_ticks_observed: observed_max_wait, bound_ticks: bound } } else { BranchDischargeResult::Violated { max_wait_ticks_observed: observed_max_wait, bound_ticks: bound, class } }; per_branch_discharge.insert(branch_id, result); } PerBranchLivenessReport { base_report, per_branch_discharge } } } 

Params::shader_proof_obligation_scope gains PerBranchFair(PerBranchFairnessBounds) alongside V10's SafetyOnly and SafetyPlusGlobalLiveness; the annotation table is loaded from the same policy file that defines the dispatch graph itself, so branch fairness class is declared once, at the same place branch selection logic lives, not duplicated in params.json.

8. Pillar #6 — Online Catalog-Adaptive IMM Hypothesis Swap (sim/ivis.rs)

Problem it closes: DataDrivenImmMultiFrameTracker (V10) selects its top-N kinetic hypotheses from the historical catalog once, at run start, and keeps that set fixed for the whole run. A single dataset can contain multiple distinct kinetic regimes across different tracks or time windows; a fixed pre-run selection can't adapt to a regime the initial ranking under-weighted.

Fix: add an online controller that watches each track's own recent per-hypothesis likelihood history and, when every currently-loaded hypothesis is fitting poorly while a catalog entry outside the loaded set would plausibly fit better, swaps it in mid-run — without disturbing the two mandatory V9 CV/CA hypotheses, which V10 already guarantees stay loaded.

// sim/ivis.rs — EXTENDED /// V10, unchanged; still the default under a static catalog-selection mode, /// and V11's tracker composes it as `base`. pub use crate::sim::ivis::{DataDrivenImmMultiFrameTracker, CatalogKineticHypothesis}; #[derive(Debug, Clone, Copy, PartialEq, Eq, serde::Serialize, serde::Deserialize)] pub enum PtvImmCatalogAdaptationMode { Static, // V10 — top-N fixed at run start OnlineSwap, // V11 — can hot-swap mid-run } pub struct HotSwapTrigger { /// A currently-loaded (non-CV/CA) hypothesis is a swap candidate once its /// trailing mean log-likelihood over `evaluation_window_ticks` falls /// below `poor_fit_log_likelihood_threshold`. pub evaluation_window_ticks: u32, pub poor_fit_log_likelihood_threshold: f64, /// A catalog candidate not currently loaded is only swapped in once its /// *retrospective* log-likelihood, computed against the same recent /// observation window, exceeds the worst currently-loaded hypothesis by /// this margin — avoiding thrashing on noise-level differences. pub swap_in_margin: f64, /// Minimum ticks between swaps for a given track, to bound churn. pub min_ticks_between_swaps: u32, } /// V11: composes V10's tracker as `base`. if let Some(last_swap) = self.per_track_last_swap_tick.get(&track_id) { if tick - last_swap < self.trigger.min_ticks_between_swaps as u64 { return result; // still cooling down from the last swap } } if let Some(swap) = self.evaluate_swap_candidate(track_id, tick) { self.apply_swap(track_id, tick, swap); } result } fn evaluate_swap_candidate(&self, track_id: TrackId, tick: u64) -> Option<PendingSwap> { let loaded = self.base.loaded_non_cv_ca_hypotheses(track_id); // V9 CV/CA excluded by construction let worst = loaded.iter() .filter(|h| self.base.trailing_log_likelihood(track_id, h.id, self.trigger.evaluation_window_ticks) < self.trigger.poor_fit_log_likelihood_threshold) .min_by(|a, b| self.base.trailing_log_likelihood(track_id, a.id, self.trigger.evaluation_window_ticks) .partial_cmp(&self.base.trailing_log_likelihood(track_id, b.id, self.trigger.evaluation_window_ticks)) .unwrap())?; let worst_ll = self.base.trailing_log_likelihood(track_id, worst.id, self.trigger.evaluation_window_ticks); let candidate = self.full_catalog.iter() .filter(|c| !loaded.iter().any(|l| l.id == c.id)) .map(|c| (c, self.base.retrospective_log_likelihood(track_id, c, self.trigger.evaluation_window_ticks))) .filter(|(_, ll)| *ll > worst_ll + self.trigger.swap_in_margin) .max_by(|(_, a), (_, b)| a.partial_cmp(b).unwrap())?; Some(PendingSwap { swap_out: worst.clone(), swap_in: candidate.0.clone(), trailing_ll_out: worst_ll, retrospective_ll_in: candidate.1, }) } fn apply_swap(&mut self, track_id: TrackId, tick: u64, swap: PendingSwap) { self.base.replace_hypothesis(track_id, &swap.swap_out, swap.swap_in.clone()); self.per_track_last_swap_tick.insert(track_id, tick); self.swap_log.push(HotSwapEvent { track_id, tick, swapped_out: swap.swap_out, swapped_in: swap.swap_in, trailing_log_likelihood_out: swap.trailing_ll_out, retrospective_log_likelihood_in: swap.retrospective_ll_in, }); } } struct PendingSwap { swap_out: CatalogKineticHypothesis, swap_in: CatalogKineticHypothesis, trailing_ll_out: f64, retrospective_ll_in: f64, } 

stats.rs gains imm_catalog_hot_swap_log: Vec<HotSwapEvent> so post-run analysis can check whether swaps clustered around genuine known regime boundaries in a labeled validation dataset, or thrashed — the latter being the signal to widen swap_in_margin or min_ticks_between_swaps.

9. Pillar #7 — Federated External Governance Authority (sim/provenance.rs)

Problem it closes: AuthorityCosignedNotaryLedger (V10) requires notary quorum and one designated external authority's co-signature for below-floor removal. That single external authority is now itself a single point of coercion or compromise for exactly the case V10 was built to protect — the same structural weakness V9's move from single-operator to notary-quorum was meant to close, recurring one layer up.

Fix: replace the single external authority with a configurable set of external authorities and its own quorum requirement, independent of (and structurally identical in spirit to, but organizationally separate from) the notary quorum.

11. Suggested Next Steps (V12 candidates)

Fiber coupling: the diffusion coefficient in DiffusionCoupledRemodelingModel is currently a single global scalar; real fascicle boundaries should attenuate coupling strength, which would need coupling weights derived from a segmented-tissue-boundary map rather than uniform face-adjacency.

Shared-factor correlation: SharedFactorCorrelationEstimator's rank k is fixed at config time; an online rank-selection criterion (e.g. tracking the eigenvalue gap) would let the model grow or shrink its factor count as the number of genuinely independent shock sources in the fleet changes over a long-running deployment.

Regime targeting: the volatility head's confidence floor is a static threshold; a calibrated, per-regime-transition-type confidence estimate (some transition types are inherently easier to predict the target of than others) would reduce unnecessary fallback-to-V10 behavior.

Trajectory frontier: the trajectory feature is still a fixed-window slope/curvature summary; a learned sequence encoder over the state history would let the model use trajectory shapes beyond what a quadratic fit can represent.

Per-branch liveness: fairness classes (Hot/Warm/Fallback) are still hand-declared; deriving them from observed historical selection frequency, with the symbolic executor flagging a mismatch between declared and observed class as a policy-authoring smell, would catch stale annotations after a scheduler policy changes.

Catalog-adaptive IMM: hot-swap decisions are currently per-track and independent; a cluster-level view (many tracks swapping toward the same catalog entry around the same tick is itself evidence of a genuine dataset-wide regime change, distinct from track-level noise) could raise swap confidence and lower min_ticks_between_swaps during a corroborated event.

Federated authority: the federated quorum threshold is currently static; a threshold that adapts with the requested removal's magnitude (a larger below-floor removal warranting a larger authority quorum) would proportion the governance friction to the risk of the specific action.
